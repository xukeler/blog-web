    # sass

    • Sass是一种CSS的预编译语言，预编译简单理解就是为编译做准备的阶段，主要是做代码文本的替换工作。最终还是要编译为Css，让浏览器识别加载运行(浏览器识别不了Sass，需要编译)。
    • 在Vue开发中项目经常会用Scss，Scss和Sass是同一个东西，只不过它们的后缀不同一个以.sass结尾,一个以.scss结尾，语法不同，Sass是以严格的缩进式语法规则来书写，不带大括号({})和分号(;)，而 Scss的语法书写和我们的Css语法书写方式非常类似。
    • Scss 提供了变量variables、嵌套nested rules、 混合mixins、 函数functions等功能，并且完全兼容 Css语法,在主题配置这篇文档有用到。
    • 使用scss在项目中引入node-sass和sass-loader，在项目中经常见到。
    • 在一般开发中用到最多的就是变量语法嵌套和@mixin,用的多掌握，用的少的知道有这个东西就行。

    # 使用变量

    • 在Scss中声明变量使用$符号,使用这个变量在Scss语句中引入。
    • 在外部声明的全局可用,在大括号内部声明的，只能在括号内部使用，在其它的大括号内也可以声明同名的变量，互不影响，和js的作用域类似。
    • 声明变量也可以引用其它的变量例如$color:$backgroundColor。
    • 变量名可以与css中的属性名和选择器名称相同，包括中划线和下划线，$link-color和$link_color其实指向的是同一个变量。
    • 默认变量值，反复声明一个变量，后面的会覆盖前面的变量，这个时候就需要一个默认的值!default，在别的文件中没有定义这个变量，这个变量就是默认值，否则就会失效。
    • 支持的数据类型。数字，1, 2, 13, 10px,字符串，有引号字符串与无引号字符串，"foo", 'bar', baz,颜色，blue, #04a3f9, rgba(255,0,0,0.5),布尔型，true, false,空值，null,数组 (list)，用空格或逗号作分隔符，1.5em 1em 0 2em, Helvetica, Arial, sans-serif,maps, 相当于JavaScript的object，(key1: value1, key2: value2)`。
    <style lang="scss">
    $color:red  !default;
    .main-container{
        $backgroundColor:black;
        color:$color;
        .menu{
            background-color:$backgroundColor;
        }
    }
    .nav-container{
         $backgroundColor:$color;
         .nav-list{
             background-color:$backgroundColor;
         }
    }
</style>

/* 编译结果 */
    .main-container{
        color:red;
    }
    .main-container .menu{
        background-color:black;
    }
    .nav-container .nav-list{
        background-color:red;
    }

    # 嵌套css

    • 在开发中我们常用的css写法，很繁琐，每写一个类的样式就要把它的父级类名加上，最后就越加越长，就降低了开发效率。
    • 使用scss嵌套css,在父级的css大括号里面写自己的样式，然后一级一级嵌套，这样不用写很多类名，可读性很高，
    • 在嵌套的css使用:hover ::before ::after可以使用父级选择器&符号实现，在大括号内部定义。
    • 群组选择器,和后代选择器> + ~使用和css一样。
        .main-container{
        color:red;
        &:hover{
            color:black;
        }
        .menu{
            background-color:green;
            .list{
                color:white
            }
        }
    }
    /* 编译结果 */
    .main-container {
      color: red;
    }
    .main-container:hover {
      color: black;
    }
    .main-container .menu {
      background-color: green;
    }
    .main-container .menu .list {
      color: white;
    }

    # 导入Scss文件

    • css有个@import来引入外部的css文件,只有执行到@import时，浏览器才会去下载其它css文件，这导致页面加载起来特别慢。
    • scss也是通过@import来引入外部的Scss文件，不同的是sass的@import规则在生成css文件时就把相关文件导入进来。这意味着所有相关的样式被归纳到了同一个css文件中，而无需发起额外的下载请求。
    • 可以省略.sass或.scss文件后缀。
    • 局部文件,以下划线_开头命名的scss文件，会被识别为局部文件，在编译的时候就不会为局部文件单独编译css，而是把这个文件当做导入。
    @import '../nav/_nav

    # 函数 function

    • Sass支持自定义函数，并能在任何属性值使用，以@function 开头定义函数，需要返回值@renturn，可以像js一样传递参数。参数名以$开头。
    • 使用在属性内部调用函数即可。
    $width:10px;
@function get-width($n) {
  @return $n * 5px + ($n - 1) * $width;
}

#sidebar { 
    width: get-width(5); 
    }

    # 控制指令 if else

    • scss的if判断需要以@标识开头，满足条件引用对应的样式，也可以使用else if
    $maxin:10px;
.main{
    @if $maxin==9px{
        font-size:12px;
    }
   @else if 5<3{
        font-size:30px
    }
    @else{
        font-size:10px
    }
}
//编译
.main {
  font-size: 10px;
}

    # 混合器@mixin

    • @mixin可以理解为定义公用的部分，变量只能定义一个属性，而在多个地方共用多个css样式，就可以用@mixin。
    • @mixin定义scss属性，通过@include来调用。
    • @mixin定义scss属性可以包含变量，也可以传参，可以设置默认值，默认值可以是变量。
    $color:red;
@mixin public{
    color:$color;
    font-size:12px;
    font-weight：600;
}
.main-container{
    background-color:green;
    @include public
}

/* 传参 */
@mixin public($color, $fontSize) {
    color:$color;
    font-size: $fontSize
}

.main-container{
    background-color:green;
    @include public(red,12px)
}

/* 默认值 */
$smallSize:10px;
@mixin public($color:red, $fontSize:$smallSize) {
    color:$color;
    font-size: $fontSize
}
.main-container{
    background-color:green;
    @include public(red,12px)
}
.nav-container{
    background-color:green;
    @include public
}

    # 继承@extend

    • 继承在正常的开发中用的不多，它和@mixin的区别是，@mixin是样式的重复使用，@extend是类的重复使用，可能会有权重问题
    • 详细参考官方文档
        .name{
        color:red
    }
    .sex{
        @extend .name;
    }
