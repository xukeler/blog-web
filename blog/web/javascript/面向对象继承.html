    # 面向对象编程

    • JavaScript是一个不完全面向对象的弱类型语言，它里面的数据都可以看成对象。JavaScript是通过原型（prototype）来实现面向对象编程。
    • 对象可以看作一系列属性和方法的集合。
    • 对象都是基于一个引用类型创建，这个引用类型可以是原生类型比如Object,Date,Array等。
    • 每个对象都有一个__proto__属性指向它的构造器原型constuctor.prototype,它会一级一级向上查找直到null为止。

    # 工厂模式

    • 工厂模式是用函数来封装，将对象返回出去。
    
function init(name, age) {
    var obj = new Object();
    obj.name = name;
    obj.age = age;
    obj.getName = function() {
        alert(this.name);
    };
    return obj;
}

var res1 = init('小明', 22);
var res2 = init('小红', 24);

    # 构造函数模式

    • 构造函数可用来创建特定类型的对象，从而定义对象的属性和方法
    • 构造函数的名称必须大写，使用必须new实例化一个对象。
    • 和工厂模式相比少了return返回的东西。
    • 缺点是每个方法属性在每个实例上都要重新创建一遍
    function Init(name, age) {
    this.name = name;
    this.age = age;
    this.getName = function() {
        alert(this.name);
    };
}

var res1 = new Init('小明', 22);
var res2 = new Init('小红', 24);
console.log(res1.name,res2.age)//小明 24

    # 原型模式

    • 创建的每个函数都有一个prototype ( 原型 ) 属性，这个属性prototype属性指向函数的原型对象。在默认情况下，所有原型对象都会自动获得一个 constructor构造函数）属性。constructor指向prototype属性所在的函数。
    • 使用原型对象的好处是可以让所有对象实例共享它所包含的属性和方法。不管new几个实例对象，调用的prototype的属性和方法都是同一个，不会重新创建。
    • 在prototype设置了属性或者方法之后，重写了默认的prototype对象，因此prototype.constructor指向就会改变，不再指向Init，如果觉得这个很重要可以再设置回来
    • 缺点是原型中所有属性是被很多实例共享的，多个实例化对象都可以设置获取到原型中的属性。
    function Init(name, age) {
    this.name = name;
    this.age = age;
    this.getName = function() {
        alert(this.name);
    };
}
Init.prototype.gender="女";
Init.prototype.array=[];
var res1 = new Init('小明', 22);
res1.array.push('1')
var res2 = new Init('小红', 24);
res2.array.push('2')
console.log(res1.array,res2.array)//["1", "2"],["1", "2"] //原型中所有属性是被很多实例共享

Init.prototype.constructor=Init;//重新设置constructor指向

    # 构造函数模式和原型模式

    • 构造函数模式用于定义实例属性，而原型模式用于定义方法和共享的属性。
    funciton Init(name, age, sex) {
    this.name = name;
    this.age = age;
    this.sex = sex;
    this.friends = [1,2];
}

Init.prototype = {
    constructor: Person,
    sayName: function() {
        alert(this.name);
    }
}

var res1 = new Init('小明', 22);
var res2 = new Init('小红', 24);

res1.friends.push(3);

alert(res1.friends); // '1,2,3';
alert(res2.friends); // '1,2';

alert(res1.friends === res2.friends); // false;
alert(res1.sayName === res2.sayName); // true;

    # 继承

    • 只有new的实例化对象才能拿到构造函数的所有的方法，包括原型对象上的。
    • 使用原型继承，缺点把私有的（构造函数内部定义的）和公共的（原型对象上定义的）全部放在了子构造函数的原型对象
    • JavaScript只有一种结构：对象。每个实例对象（ object）都有一个私有属性（称之为 __ proto __）指向它的构造函数的原型对象（prototype）。该原型对象也有一个自己的原型对象( __ proto __) ，层层向上直到一个对象的原型对象为 null。根据定义，null没有原型，并作为这个原型链中的最后一个环节。由此就形成一条链，这条链就叫做原型链。
    function Person(){
    this.name='小明';
}
function Study(){
    this.gender='男'
}
Person.prototype=new Study();
Person.prototype.constructor=Person //Person.prototype.constructor 构造器指向Study，需要修改指向Person
var res=new Person();
console.log(res.gender)
