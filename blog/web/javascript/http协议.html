    # http协议

    • HTTP 是基于 TCP/IP 协议的应用层协议（超文本传输协议）,规定了客户端和服务器之间的通信格式，简单来说就是客户端和服务端进行数据传输的一种规则，常见的get post delete请求都是http协议的命令。

      • 应用层协议。
        • 简单理解就是互联网通信的实现，分成好几层，有的层传输，有的层负责应用，每层的功能都不相同。
        • 应用层只是其中一层， 应用层的作用就是规定应用程序的数据格式,例如通过TCP协议传输的一些邮件信息，不同的协议处理这些发的不同的数据，这些协议构成了应用层。
      • TCP/IP协议
        • TCP/IP传输协议，即传输控制/网络协议，也叫作网络通讯协议。它是在网络的使用中的最基本的通信协议。TCP/IP传输协议对互联网中各部分进行通信的标准和方法进行了规定。并且，TCP/IP传输协议是保证网络数据信息及时、完整传输的两个重要的协议。

    • http协议每次连接只处理一个请求，完成之后就断开连接。

    • http消息结构，请求行，请求头，空行，请求数据四个部分组成，Http响应也由四个部分组成，状态行、消息报头、空行和响应正文。

    • http协议的8种请求类型,虽然 HTTP 的请求方式有 8 种，但是我们在实际应用中常用的也就是 get 和 post，其他请求方式也都可以通过这两种方式间接的来实现。

      • OPTIONS允许客户端查看服务器的性能。
      • GET请求指定的页面信息，并返回实体主体。
      • POST向指定资源提交数据进行处理请求（例如提交表单或者上传文件）。数据被包含在请求体中。POST请求可能会导致新的资源的创建和/或已有资源的修改。
      • PUT从客户端向服务器传送的数据取代指定的文档的内容。
      • DELETE请求服务器删除指定的页面。
      • TRACE回显服务器收到的请求，主要用于测试或诊断。
      • HEAD类似于 GET 请求，只不过返回的响应中没有具体的内容，用于获取报头
      • CONNECT HTTP/1.1 协议中预留给能够将连接改为管道方式的代理服务器。

    • http请求信息

      • 请求的url例如localhost://8080
      • 请求方法 例如get post delete option put等
      • 请求头header
        • Accept 指定客户端能够接收的内容类型。
        • Accept-Charset浏览器可以接受的字符编码集，常用的就是utf-8。
        • Accept-Language浏览器可接受的语言，一般就是en/zh
        • Accept-Ranges 可以请求网页实体的一个或者多个子范围字段，一般用于大文件的上传。
        • Authorization http授权的授权证书类型。
        • Cache-Control 指定请求和相应的缓存机制。
        • Connection 表示是否需要持久连接。
        • Cookie 用于客户端和服务端来回通信。
        • Content-Length请求内容的长度。
        • content-type 请求的实体对应的MIME信息。
        • User-Agent 客户端标识

    • http请求响应信息

      • Allow服务器支持哪些请求方法（如GET、POST等）。
      • Content-Encoding文档的编码
      • Content-Length表示内容长度
      • Content-Type表示后面的文档属于什么MIME类型。Servlet默认为text/plain，但通常需要显式地指定为text/html。由于经常要设置Content-Type，因此HttpServletResponse提供了一个专用的方法setContentType。
      • Date当前的GMT时间。
      • Expires什么时候认为文档已经过期，从而不再缓存它
      • Last-Modified文档的最后改动时间
      • Refresh表示浏览器应该在多少时间之后刷新文档
      • Server服务器名字
      • Set-Cookie设置和页面关联的Cookie

    • Http状态码

      • 200 请求成功
      • 301 重定向
      • 404 请求的资源（网页等）不存在
      • 500 内部服务器错误
      • 以1开头的很少，服务器收到请求，需要请求者继续执行操作。
      • 以2开头表示请求成功
      • 以3开头一般都是重定向
      • 以4开头一般是客户端错误
      • 以5开始是服务器错误
