    # EventLoop

    • EventLoop是一种解决javaScript单线程运行时不会阻塞的一种机制，在node和浏览器中都存在。
    • javaScript单线程运行时从任务队列中读取事件，这个过程是循环不断的，所以整个的这种运行机制又称为Event Loop事件循环。

    # 浏览器中js的事件循环机制

    • 在浏览器中js执行是分为同步任务和异步任务，同步就是程序一直从上向下执行，每一步的执行都能得到预期的结果，异步就是程序从上向下执行，在执行到某一步的时候不能马上得到预期的结果，需要等待一会，比如ajax setTimeout等。
    • 浏览器中异步任务分为两类宏任务和微任务(microtask)，微任务一般就是promise await，宏任务(macrotask)一般是setTimeout setInterval ajax Dom事件
    • js在执行程序的时候，先执行同步的程序，遇到异步的程序，会将异步的事件放在另外一个事件队列，不会马上执行，而是等执行栈在执行完同步任务后，查看执行栈是否为空，如果执行栈为空，就会去检查微任务再执行事件队列。当微任务的事件队列执行的过程中产生新的微任务，那么会加入到队列的末尾，也会在这个周期被调用执行，当微任务队列全部执行完毕，再去调用宏任务。宏任务执行完毕会再次调用微任务队列，以此循环。
    1.执行全局Script同步代码，这些同步代码有一些是同步语句，有一些是异步语句（比如setTimeout等）；
2.全局Script代码执行完毕后，调用栈Stack会清空；
3.从微队列microtask queue中取出位于队首的回调任务，放入调用栈Stack中执行，执行完后microtask queue长度减1；
4.继续取出位于队首的任务，放入调用栈Stack中执行，以此类推，直到直到把microtask queue中的所有任务都执行完毕。注意，如果在执行microtask的过程中，又产生了microtask，那么会加入到队列的末尾，也会在这个周期被调用执行；
5.microtask queue中的所有任务都执行完毕，此时microtask queue为空队列，调用栈Stack也为空；
6.取出宏队列macrotask queue中位于队首的任务，放入Stack中执行；
7.执行完毕后，调用栈Stack为空；
8.重复第3-7个步骤；
9.重复第3-7个步骤；

    # 面试题示例

    • 在面试的过程中遇到一道面试题，能够说明eventloop事件循环机制。
    • 下列代码的打印顺序。
    async function async1() {
    console.log('async1 start');
    await async2();
    console.log('async1 end');
}
async function async2() {
    console.log('async2');
}
console.log('script start');
setTimeout(function() {
    console.log('setTimeout');
}, 0)
async1();
new Promise(function(resolve) {
    console.log('promise1');
    resolve();
}).then(function() {
    console.log('promise2');
});
console.log('script end');

    • 宏任务队列中，只有一个script(整体代码)任务,先运行这个宏任务，宏任务在执行的时候遇到异步的操作会注册事件到微任务队列或宏任务队列。
    • 所以先打印script start,async1 start,async2,promise1,script end,需要注意的是await的执行，从字面意思上看await就是等待，await等待的是一个表达式，这个表达式的返回值可以是一个promise对象也可以是其他值。很多人以为await会一直等待之后的表达式执行完之后才会继续执行后面的代码，实际上await是一个让出线程的标志。await后面的表达式会先执行一遍，将await后面的代码加入到microtask中，然后就会跳出整个async函数来执行后面的代码。promise里面的回调函数也是同步执行，但是.then .catch是异步的。
    • 当同步的执行完就开始执行微任务promise的.then，和await，按照先入先出的原则，先打印async1 end,promise2。
    • 当微任务执行完毕就开始执行宏任务setTimeout，打印setTimeout。
