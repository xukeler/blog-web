    # js作用域

    • 作用域简单来说就是变量的使用范围，js的作用域分为全局作用域，局部作用域，es6新增了块级作用域。
    • 全局作用域就是能在程序的任何地方都能使用，作用全局，声明也必须全局声明，不能用{}嵌套，也不能写在函数里面()=>{}。
    • 局部作用域就是在一部分地方使用，一般是指在函数内部声明的变量，那么它的作用范围只能在函数内部访问，外部是访问不到的。
    • 块级作用域是指被{}包围的，一般指被if(){},for(){}包围的语句，在这里面声明的变量，它的作用范围就在{}的内部。

    # 全局作用域

    • 如果使用var声明，变量会放在window对象下面，通过window.变量名能够访问到。
    • 如果使用let cosnt声明的变量和常量，虽然能够全局访问，但是不会放在window下面。
    • 在es6的描述为什么let const声明的变量或常量没有放在window上，因为var命令和function命令声明的全局变量，依旧是顶层对象的属性(window),另一方面规定，let命令、const命令、class命令声明的全局变量，不属于顶层对象的属性。也就是说，从 ES6开始，全局变量将逐步与顶层对象的属性脱钩,也就是为什么不会在window上。
    
var a=1;
let b=2;
const c=3;
console.log(a,window.a,b,window.b,c,window.c)  //1 1 2 undefined 3 undefined

    # 局部作用域

    • 局部作用域一般在函数内部声明，能够在函数内部任意使用，如果想在函数外部访问到函数内部的变量就涉及到闭包。
    • 不管里面再嵌套几个函数都是可以访问的，这个就涉及到作用域链。
    • 作用域链的概念就是在访问一个变量的时候，先在当前作用域进行查找，没有找到，就会向上的作用域进行查找，一级一级的查找，直到找到或找不到位置，一级一级的作用域查找就构成作用域链。
    function getData() {
  var _name="ming";
  console.log(_name);  //ming
  function getName() {
    console.log(_name)  //ming
  }
  getName()
} 
getData()
console.log(_name)  //_name is not defined

    # 块级作用域

    • 使用let cosnt在if for语句内声明变量，在外面是访问不到的，但是在if for声明的函数依旧是放在window上面的,块级作用域内的函数声明语句的行为对外类似于var ，对内类似于 let。
    • 在函数内部访问函数名，对变量和函数名同名的变量赋值是无效的，因为在函数内部访问函数名，相当于函数名是一个常量，不能对常量进行赋值操作。

console.log(a,c)  // c is not defined, a is not defined
