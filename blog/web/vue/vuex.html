    # vuex

    • vuex是vue组件状态存储管理工具。在项目中store.js中配置，需要下载安装vuex,使用脚手架工具会配置好。
    • vuex包含五个模块state，getters,mutations,actions,modules。
    • 一般在开发大型的应用时会用到。小项目建议不要使用。
    • 官方文档

    # state

    • state是用来状态存储,在main.js引入后，就可以在组件里通过this.$store.state.状态名来获取状态，也可以通过辅助函数mapState来获取。
    • mapState要在计算属性中获取,使用直接this.状态名使用。
    import { mapState } from 'vuex'
mounted(){
    console.log(this.$store.state.count)
}
computed:{
    ...mapState(["count"]),
}

    # mutations

    • 更改state的唯一途径，mutations里面写的是修改state的方法，接收state作为第一个参数，也可以传入额外的参数。
    • 在组件中调用mutations的方法一种this.$store.commit('方法名'，参数，参数......)，也可以以对象的方式。也可以通过辅助函数mapMutations。
    • 注意mutations必须是同步的。
    //vuex
mutations:{
    setData(state,count){
        state.count=count
    }
}

//组件
import { mapMutations } from 'vuex'

methods:{
    ..mapMutations['setData']
    change(){
        this.setData(20)//
        //或
        this.$store.commit('setData',20)
        //或
         this.$store.commit({
             type:"setData",
             count:20
         })
    }
}

    # getters

    • getters是vuex的计算属性,当state改变时也能触发getters。
    • getters默认接收state为第一个参数。
    • 组件内使用this.$store.getters.计算属性名,或者辅助函数mapGetters在计算属性中调用。
    //vuex
getters:{
    getData(state){
        return state.dataArray.filter((item)=>{
            return item>5
        })
    }
}

//组件
import { mapGetters } from 'vuex'

computed:{
    ..mapGetters['getData']
}
mounted(){
    console.log(this.getData)
    console.log(this.$store.getters.getData)
}

    # actions

    • actions可以执行异步操作的mutations，例如发起网络请求。
    • actiona 函数接受一个与 store 实例具有相同方法和属性的 context 对象，因此你可以调用 context.commit提交一个 mutation，或者通过context.state 和context.getters 来获取 state 和 getters。
    • 使用辅助函数mapActions调用。
    mutations:{
    setData(state,count){
        state.count=count
    }
}
actions: {
  getData (context) {
    //异步操作
    context.commit('setData',20)
  }
}

//组件

import { mapActions } from 'vuex'

methods:{
    ...mapActions['getData']
},
mounted(){
   this.getData() 
}

    # Module

    • Module是将整个的store分割成一个个小模块，每个模块拥有自己的 state、mutation、action、getter。
    • 不怎么常用
    const moduleA = {
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... },
  getters: { ... }
}

const moduleB = {
  state: () => ({ ... }),
  mutations: { ... },
  actions: { ... }
}

const store = new Vuex.Store({
  modules: {
    a: moduleA,
    b: moduleB
  }
})

    # 辅助函数

    • 通常使用state，或者mutation都需要this.$store.state或者this.$store.commit来使用，但是有时候调用太多的状态，这样使用还是有些麻烦，所以可以借助vuex的辅助函数来解决这个问题。
    • 通过辅助函数mapState、mapActions、mapMutations，把vuex.store中的属性映射到vue实例身上，这样在vue实例中就能访问vuex.store中的属性了。 使用时需要在组建内引入

    注意

    注意事项，映射的名称不要和当前组件的data，methods名称发生冲突，否则会被覆盖。

    import {mapState,mapMutations,mapActions,mapGetters} from "vuex"

    • state的辅助函数需要映射到计算属性中computed，映射的名称一定要相同，然后就可以通过this访问到state。
      computed:{
        ...mapState(["name"]),//name和vuex中的state保持一至。
    },
methods：{
  click(){
    console.log(this.name)//访问到vuex的name

    # state刷新消失

    • 在项目中会遇到这种问题，同过vuex进行公共的状态管理，在父级页面设置的state,跳转到子页面的时候，是可以拿到state的初始值，但是当子页面刷新的时候，vuex的state赋值会恢复到父页面没有修改的状态（父级页面的修改操作无效）。 所以解决vuex刷新消失的问题可以通过本地存储来解决，思路就是，刷新的时候判断本地存储有没有存储的键值对（赋值的vuex），如果有，就通过mutation就把本地存储的键值对赋值给state，如果没有，就证明用户是第一次进来，存一个本地存储的键值对。代码如下。

    提示

    注意事项，修改vuex的state的唯一途径就是通过mutation修改。

     state: {
    type: '',
  },
 mutations: {
    change(state, val) {
      if (val) {
        state.type = val;
      }
    }
  },
