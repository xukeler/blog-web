    # vue开发插件发布到npm

    • 开发经常会遇到这种场景，一般的ui框架满足不了需求，所以要自己封装组件，在多个项目中都有用到，例如checkbox组件等，一般是复制粘贴过去，但是这种不太好，如果是全程参与多个项目开发的人，他知道哪个项目里面有可以复用的组件，直接粘贴过来就能用，但是对于新来的员工，他是不知道，新员工开发的时候可能会找插件或者重新封装，这个就比较浪费资源了，而且还不统一。所以可以把跨项目通用的组件封装成一个vue插件，上传到npm，每次要用直接npm install安装使用，再维护一个文档记录npm插件。

    # 开发

    • 开发一个vue插件合正常开发组件一样，使用vue-cli脚手架搭建一个项目，新建一个总目录，再创建一个子目录，在子目录创建一个组件，正常测试功能都正常，在总目录和子目录内分别添加一个index.js文件，总目录npm init添加一个package.json文件，一个README.md文件。
    • 目录结构，也可以增加其它子目录没有限制。
      • checkBox总目录
        • lib子目录
          • XXXXX.vue组件
          • index.js(暴露给Vue一个install方法,install方法接受两个参数，一个vue示例，一个option,在install方法内把组件注册成全局组件)
        • index.js(导出lib文件夹的index.js,目的是下载后可以import使用)。
        • package.json记录版本号，插件名等信息
        • README.md插件说明文件

    # 上传到npm

    • 上传到npm首先要注册一个npm账号，
    • 然后在插件总目录用命令行工具执行npm login，输入用户名，密码，邮箱，验证邮箱后就登录成功，
    • 执行npm publish命令上传到npm，上传成功后就可以在npm搜到了。

    注意

    插件名不要和别人的重复，使用合法报名，否则上传不成功。

    # 插件代码示例

    • checkBox/index.js
    module.exports=require('./lib')
    1
    • checkBox/lib/index.js
    // checkBox.vue写我们的组件
import checkBox from "./checkBox.vue"

// 定义我们的插件
const myPlugin = {    
    // 该插件有一个install方法
    // 方法的第一个参数是传入的Vue，第二个参数可以插件的自定义参数
    install (Vue, options) {
        // 将其注册为vue的组件，'checkBox'是组件名，checkBox是我们开发的组件
        Vue.component('checkBox', checkBox)    
    }
}
// 最后将插件导出，并在main.js中通过Vue.use()即可使用插件
export default myPlugin

    # 使用

    • 使用先npm install下载安装，在main.js文件import checkBox from 'checkbox-1'引入。
    • Vue.use(checkbox)，安装插件，直接在页面使用即可。
