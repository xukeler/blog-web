# TypeScript简介

  • TypeScript是由微软发明，TypeScript具有类型系统，且是JavaScript的超集。 它可以编译成普通的JavaScript代码,识别使用普通的JavaScript语法。 TypeScript支持任意浏览器，任意环境，任意系统并且是开源的。 TypeScript目前还在积极的开发完善之中，不断地会有新的特性加入进来。
  • 使用TypeScript主要是为了方便项目的维护，以及减少代码的出错率。

# 下载TypeScript

  • 使用TypeScript需要使用命令行工具npm下载TypeScript。
  • 一般TypeScript都在项目里面搭建好了，只需要写就可以自动转换为js。
npm install typescript -g

  • 在引入js文件的同级目录中创建一个.ts的文件，和html页面引入的js文件同名，但不用创建js文件，js的代码都写在ts文件里，在命令行窗口通过命令tsc将.ts文件转换为.js文件,转换完成之后目录中会出现一个.js文件。

注意，如果要使用es6的语法，下载babel

# 元组Tuple

  • 元祖Tuple，元组类型表示一个已知元素数量和类型的固定长度数组，各元素的类型可以不相同，但是在给数组赋值的时候必须和定义的类型一样，数组长度必须一样，必须按照指定数据类型的方法操作，否则会报错。

console.log(arr)//['string',10.{name:"222"}]

# any

  • 任意值any在编程阶段还不清楚类型的变量指定一个类型，在编译时可选择地包含或移除类型检查，定义any和其他数据类型有很大的区别，定义数据类型为any之后，变量再次赋值不同数据类型，变量的数据类型也会改变（这在JavaScript里面是很正常的，但是在TypeScript必须设置any），定义为其他数据类型的变量，再次赋值为不同数据类型，会报错。

let num:any;
num=123;
num="123";//都不会报错
let count:number=5;//number
count=true;
console.log(typeof num)//number
console.log(typeof count)//报错
num=true
console.log(typeof num)//boolean

# Object

  • object类型和js的object是一样的，声明object类型需要对每个属性声明，因为ts检查比较严格，在对对象赋值的时候，多一个属性，少一个属性都会报错，所以需要设置[attrName:string]:any，意思是除了必须赋值的属性，还可以额外添加没有声明的字符串类型的属性名的属性，设置any是允许各种类型的属性值，也可以设置number string等，但是属性值就固定了。
let obj:{name:string,gender:string,age:number}  //定义对象属性类型
obj={
    name:"sfsdf",
    gender:"nv",
    age:18,
    school:"rerer" //多了school 报错
}


let obj:{name:string,gender:string,age:number,[attrName:string]:any}  //attrName是自己定义的，可以随便改
obj={
    name:"sfsdf",
    gender:"nv",
    age:18,
    school:"rerer",
    home:"fsdfs"   //不会报错
}

# 类型断言

  • 类型断言其实就是告诉解析器实际的变量类型，当变量的类型自己清楚，但是解析器不清楚，而且报错的时候，就需要类型断言指定数据类型，例如unknown。
  • 两种写法，一种变量 as 类型，一种<类型> 变量名。
let s:string;
s="4555";
let n:unknown;
n="666";
s=n //报错
s=n as string    //指定string类型
//或
s=<string>n

# 类型声明

TypeScript的变量声明和JavaScript的变量声明都是通过关键字var，let，const来声明，这些关键字声明有一定的区别。

  • let声明的变量存在块级作用域，这点和const一样，但是let不存在预解析（变量的声明提前），而var是存在的，且var声明的只有全局作用域（作用域指变量的使用范围）和局部作用域，可以声明但不赋值，而const不可以，const只能声明常量（常量值不变，一直是固定的），且声明必须赋值。
  • TypeScript可以指定变量的数据类型。
  • 定义变量和赋值同时进行就不用定义类型，会自动识别类型，再对变量赋值其它类型会报错。
  • 可以同时对一个变量声明多种类型以|隔开。
  • 函数传递的参数也可以定义类型，返回值也可以定义类型，一旦定义了类型，定义了几个参数就要传递几个参数，多传少传都不可以。
  • 函数的返回值类型定义放在括号后面定义。
let num:number=10;
console.log(num)//10 number类型
//声明多种类型
let good:number|string;
good=1;
good="456"
//函数定义参数类型，返回值类型
function sum(num:number,cunt:number):number{
    return num+cunt
}
sum(1,2)

# 接口

  • 接口的作用就是为代码定义一些规则，代码在执行的时候，必须按照这些规则来，使用interface关键字定义接口。
  • 使用接口Interfaces来定义对象的类型
  • 接口可以用来定义一个类class的结构，定义类中有哪些属性和方法，也可以当作类型声明使用。
  • 接口只能定义结构而不能考虑实际的值，定义的所有属性都不能有实际的值。
interface SetIn{
    num:number,
    age:string
}
let obj:SetIn;
obj={
    num:4,
    age:"r4er",  //多加属性或少加属性都报错
}
//或
interface label {name:string};//定义一个label接口，它表示一个必须有一个属性名为`name`，属性值数据类型为`string`的对象。
function set(nameval:label){//nameval是参数名可以任意写，表示参数使用label接口
    console.log(nameval.name)
}
let obj={name:"ddddd"}//传入指定的参数，否则会报错
set(obj)



//定义类结构
interface typeSet{
    name:string;
    age:number;
    setName(name:string):void;
}
class People implements typeSet{
    name:string;
    age:number;
    constructor(name:string,age:number){
        this.name=name;
        this.age=age;
    }
    setName(name:string):string{
        this.name=name
        return this.name
    }
}
let res=new People("Ming",30)
console.log(res.setName("Fang")) //Fang

# 泛型

  • 泛型是在类型不明确的时候，定义一个变量表示类型，泛型一般用<自定义类型名>表示，可以定义多个。
  • 泛型可以使用extends 从接口继承。
  • 泛型也可以在类中使用。
function fn<Da,Er>(name:Da,age:Er):Da{
    console.log(age)
    return name
}
console.log(fn("Ming",18))


//extends
interface profile{
    name:string;
    age:number
}
function fn<Da extends profile>(obj:Da):Da{
    return obj
}
console.log(fn({name:'333',age:22}))  //{name: "333", age: 22}


//class
class People<Da>{
    name:Da;
    constructor(name:Da){
        this.name=name;
        console.log(this.name)  //Ming
    }

}
let res=new People<string>("Ming")
