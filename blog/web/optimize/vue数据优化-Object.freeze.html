    # vue数据优化-Object.freeze

    • Object.freeze()是es5提出的一个方法，MDN解释这个方法可以冻结一个对象。一个被冻结的对象再也不能被修改；冻结了一个对象则不能向这个对象添加新的属性，不能删除已有属性，不能修改该对象已有属性的可枚举性、可配置性、可写性，以及不能修改已有属性的值。此外，冻结一个对象后该对象的原型也不能被修改。freeze()返回和传入的参数相同的对象,只是返回的对象不能修改。
    • Object.freeze()它的作用是冻结对象，那和vue数据优化有什么关系？这个是重点。
    • 举例有一个活动页，这个活动页有一大长串列表，列表的内容是写死的，也不用请求接口，实现这个列表的方式有很多种，那种效率最高，性能最好。
      • 方案1.普通开发，列表的每一项都在页面写个标签，这样虽然能解决问题，但是效率不高。
      • 方案2，使用vue v-for在data定义一个数组，在页面通过v-for渲染，这个效率提高了，但是定义的数组后面不会有改动，只是起展示的作用，但是vue会给这个数据添加getter setter属性，这样就造成了资源浪费。所以怎么让数据即能v-for遍历显示，然后又不用添加getter setter属性，正好可以利用Object.freeze()实现。
    • vue对Object.freeze()做了处理，只要是冻结了的对象，就不会添加getter setter属性。
