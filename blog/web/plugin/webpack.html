    # webpack

    • 一个现代JavaScript应用程序的静态模块打包器，只对js进行处理，其他类型文件需要配置loader或者插件进行处理，将各个依赖文件进行梳理打包，形成一个JS依赖文件。
    • 将前端的静态资源js,less,scss,png,css,json，作为模块处理，打包成对应的静态资源bundle。

    # 为什么要用webpack

    • 因为正常开发各个依赖文件的关系难以梳理，耦合程度较高，代码难以维护,把所有依赖包都打包成为一个js文件（bundle.js）文件，会有效降低文件请求次数，一定程度提升性能。
    • 能够随时下载使用或卸载插件plugin,将es6语法转移es5，less sass语法转义css

    # 模块化开发

    • 将复杂的功能分割成隔离的小模块，每个模块能实现对应的功能，这样避免变量污染，命名冲突。提高代码复用率，提高了可维护性，方便依赖关系管理。
    • webpack就是利用模块化加载打包。

    # webpack 的五个模块

    • Entry 入口，以哪个文件为入口打包，分析构建内部的各种依赖。
    • Output输出，打包后的资源输出的地方。
    • Loader加载器，处理一下非js文件，例如scss，css，less，webpack自身只能识别js。
    • Plugins插件，加载一些实现特定功能的插件，执行范围更广的任务，例如压缩代码，去除console。
    • Mode模块，使用那种模式的配置，配置有开发模式development，线上模式production。

    # entry

    • 单入口./src/index.js，单入口打包形成一个chunk。 输出一个 bundle 文件。此时 chunk的名称默认是 main。
    • 多入口是一个数组['./src/index.js', './src/add.js']，所有入口文件最终只会形成一个chunk，输出出去只有一个bundle 文件。
    • 多入口是一个object,有几个入口文件就形成几个 chunk，输出几个 bundle文件，此时 chunk的名称是 key 值
    
entry: {
  // 最终只会形成一个chunk, 输出出去只有一个bundle文件。
  index: ['./src/index.js', './src/count.js'], 
  // 形成一个chunk，输出一个bundle文件。
  add: './src/add.js'
}

    # 使用webpack简单打包一个文件

    • 首先全局安装webpack webpack-cli，npm init获取package.json文件。

    • 创建一个入口文件index.js，和一个打包输出目录build，在入口文件index.js输入简单的一个console。

    • 执行命令webpack ./src/index.js -o ./build --mode=production 打包到线上环境，在build目录下会有一个打包好的main.js。

    • 执行命令webpack ./src/index.js -o ./build --mode=development 打包到开发环境。

    • 在index.html目录引入打包好的main.js，就能看到控制台输出的内容。

    • webpack能够编译打包js和 json文件,而不能直接编译打包其它类型的文件。

    • 开发环境打包的文件和生产环境打包的文件区别就是生产环境会压缩代码。

    • 能将 es6 的模块化语法(import)转换成浏览器能识别的语法,不能将 js 的 es6 基本语法转化为 es5 以下语法。

    • 目录文件

    # 配置样式loader

    • webpack的配置都在webpack-config.js中进行配置。
    • loader 一般配置的是样式的应用例如css,less,scss。
    • 都用到的两个loader一个是style-loader创建style将样式插入其中，再插入到head标签中。一个是css-loader，将css文件变成commonJS模块加载js中。
    • 配置less还需要less，less-loader。
    • 配置scss需要下载sass，node-sass，sass-loader。
    //webpack 的配置文件
const {resolve} =require("path")
module.exports={
    entry:"./src/index.js",  //入口文件
    output:{
        filename:"build.js",   //输出文件名称
        path:resolve(__dirname,"build")  //打包输出目录，__dirname 当前文件所在的根目录
    },
    module:{     //加载loader css-loader  scss-loader  less-loader
        rules:[
            {
                test:/\.css$/,
                use:[
                    'style-loader',
                    'css-loader'
                ]
            },
            {
                test:/\.less$/,
                use:[
                    'style-loader',
                    'css-loader',
                    'less-loader' 
                ]
            },
            {
                test:/\.scss$/,
                use:[
                    'style-loader',
                    'css-loader',
                    'sass-loader' ,
                ] 
            }
        ]
    },
    plugins:[],//插件配置
    mode:"development"     //开发模式
}

    # 打包导出html

    • 打包将js和css资源引入到一个index.html文件中，用到html-webpack-plugin插件，自动复制index.html,并且将css和js倒入其中。
    • npm i html-webpack-plugin，下载
    
//webpack 的配置文件
const HtmlWebpackPlugin = require("html-webpack-plugin")
const {resolve} =require("path")
module.exports={
    entry:"./src/index.js",  //入口文件
    output:{
        filename:"build.js",   //输出文件名称
        path:resolve(__dirname,"build")  //打包输出目录，__dirname 当前文件所在的根目录
    },
    module:{     //加载loader css-loader  scss-loader  less-loader
        rules:[
            {
                test:/\.css$/,
                use:[
                    'style-loader',
                    'css-loader'
                ]
            },
            {
                test:/\.less$/,
                use:[
                    'style-loader',
                    'css-loader',
                    'less-loader' 
                ]
            },
            {
                test:/\.scss$/,
                use:[
                    'style-loader',
                    'css-loader',
                    'sass-loader' ,
                ] 
            }
        ]
    },
    plugins:[
        new HtmlWebpackPlugin({
            //自动复制这个html， 将css和js导入
            template:'./index.html'
        })
    ],//插件配置
    mode:"development"     //开发模式
}

    # 加载图片和其它资源

    • 加载图片资源使用url-loader，html-withimg-loader或者替换成html-loader,因为我在做的时候发现html-loader在html引入的图片路径无法打开图片，所以替换成html-withimg-loader。
    • 在html和css引入图片资源处理不一样。
    • 加载其它资源除了css，html，less，scss，js等，比如字体资源使用file-loader。
    • npm i url-loader html-withimg-loader file-loader -D 。下载依赖。
    //webpack 的配置文件
const HtmlWebpackPlugin = require("html-webpack-plugin");
const {resolve} =require("path")
module.exports={
    entry:"./src/index.js",  //入口文件
    output:{
        filename:"build.js",   //输出文件名称
        publicPath:"./",
        path:resolve(__dirname,"build")  //打包输出目录，__dirname 当前文件所在的根目录
    },
    module:{     //加载loader css-loader  scss-loader  less-loader
        rules:[
            {
                test:/\.css$/,
                use:[
                    "style-loader",
                    'css-loader'
                ]
            },
            {
                test:/\.less$/,
                use:[
                    'style-loader',
                    'css-loader',
                    'less-loader' 
                ]
            },
            {
                test:/\.scss$/,
                use:[
                    'style-loader',
                    'css-loader',
                    'sass-loader' ,
                ] 
            },
            {
                test:/\.(png|jpg|jpeg|svg|gif)$/,
                loader:"url-loader",
                options:{
                    esModule:false,
                    limit: 8 * 1024,
                }
            },
            {
                test: /\.(htm|html)$/i,
                loader: 'html-withimg-loader'
            },
            {
                exclude:/\.(css|js|html|scss|less)$/,
                loader:"file-loader"
            }
        ]
    },
    plugins:[
        new HtmlWebpackPlugin({
            //自动复制这个html， 将css和js导入
            template:'./index.html'
        })
    ],//插件配置
    mode:"development"     //开发模式
}

    # 热更新和跨域代理

    • 在开发项目中,频繁的启动项目在浏览器上查看效果是一件很烦人的事，尤其是在修改css样式上，所以如果能修改后自动打开浏览器并刷新，能提高开发效率。
    • 在webpack使用webpack-dev-server来实现热更新。
    • npm i webpack-dev-server -D,启动热更新npx webpack-dev-server，webpack5.0启动命令webpack serve。
    • 跨域要使用proxy来配置代理。

    提示

    可能会遇到 webpack-cli和webpack-dev-server版本冲突的问题。

    //webpack 的配置文件
const HtmlWebpackPlugin = require("html-webpack-plugin");
const {CleanWebpackPlugin}=require("clean-webpack-plugin")
const {resolve} =require("path")
module.exports={
    entry:"./src/index.js",  //入口文件
    output:{
        filename:"build.js",   //输出文件名称

        path:resolve(__dirname,"build")  //打包输出目录，__dirname 当前文件所在的根目录
    },
    target: "web",
    module:{     //加载loader css-loader  scss-loader  less-loader
        rules:[
            {
                test:/\.css$/,
                use:[
                    "style-loader",
                    'css-loader'
                ]
            },
            {
                test:/\.less$/,
                use:[
                    'style-loader',
                    'css-loader',
                    'less-loader' 
                ]
            },
            {
                test:/\.scss$/,
                use:[
                    'style-loader',
                    'css-loader',
                    'sass-loader' ,
                ] 
            },
            {
                test:/\.(png|jpg|jpeg|svg|gif)$/,
                loader:"url-loader",
                options:{
                    esModule:false,
                    limit: 8 * 1024,
                }
            },
            {
                test: /\.(htm|html)$/i,
                loader: 'html-withimg-loader'
            },
            {
                exclude:/\.(css|js|html|scss|less)$/,
                loader:"file-loader"
            }
        ]
    },
    plugins:[
        new HtmlWebpackPlugin({
            //自动复制这个html， 将css和js导入
            template:'./index.html'
        }),
        // new CleanWebpackPlugin()
    ],//插件配置

    devServer: {
      // 运行代码所在的目录
      contentBase: resolve(__dirname, 'build'),
      // 监视contentBase目录下的所有文件，一旦文件变化就会reload
      watchContentBase: true,
      watchOptions: {
        // 忽略文件
        ignored: /node_modules/
      },
      // 启动gzip压缩
      compress: true,
      // 端口号
      port: 5000,
      // 域名
      host: 'localhost',
      // 自动打开浏览器
      open: true,
      // 开启HMR功能
      hot: true,
      // 不要显示启动服务器日志信息
      clientLogLevel: 'none',
      // 除了一些基本信息外，其他内容都不要显示
      quiet: true,
      // 如果出错了，不要全屏提示
      overlay: false,
      // 服务器代理，--> 解决开发环境跨域问题
      proxy: {
        // 一旦devServer(5000)服务器接收到/api/xxx的请求，就会把请求转发到另外一个服务器3000
        '/api': {
          target: 'http://localhost:3000',
          // 发送请求时，请求路径重写：将/api/xxx --> /xxx （去掉/api）
          pathRewrite: {
            '^/api': ''
          }
        }
      }
    }
    mode:"development"     //开发模式
}

    # 样式link标签插入head

    • 配置css使用style-loader样式就会以style标签的形式插入页面，现在要把css样式以link标签的形式插入head内。
    • 使用mini-css-extract-plugin插件，就会把css样式以link标签的形式插入。
    • 输出的css配置文件名filename，就能输出到指定目录。
    //webpack 的配置文件
const HtmlWebpackPlugin = require("html-webpack-plugin");
const {CleanWebpackPlugin}=require("clean-webpack-plugin")
const MiniCssExtractPlugin =require("mini-css-extract-plugin")
const {resolve} =require("path")
module.exports={
    entry:"./src/index.js",  //入口文件
    output:{
        filename:"build.js",   //输出文件名称
        publicPath:"/",
        path:resolve(__dirname,"build")  //打包输出目录，__dirname 当前文件所在的根目录
    },
    target: "web",
    module:{     //加载loader css-loader  scss-loader  less-loader
        rules:[
            {
                test:/\.css$/,
                use:[
                    MiniCssExtractPlugin.loader,
                    'css-loader'
                ]
            },
            {
                test:/\.less$/,
                use:[
                    'style-loader',
                    'css-loader',
                    'less-loader' 
                ]
            },
            {
                test:/\.scss$/,
                use:[
                    'style-loader',
                    'css-loader',
                    'sass-loader' ,
                ] 
            },
            {
                test:/\.(png|jpg|jpeg|svg|gif)$/,
                loader:"url-loader",
                options:{
                    esModule:false,
                    limit: 8 * 1024,

                }
            },
            {
                test: /\.(htm|html)$/i,
                loader: 'html-withimg-loader'
            },
            {
                exclude:/\.(css|js|html|scss|less)$/,
                loader:"file-loader",
                options:{
                    outputPath:"js"    //输出到js文件夹
                }
            },
            {
                test:/\.(png|jpg|gif|svg|jpeg)$/,
                loader:"file-loader",
                options:{
                    outputPath:"images"  //输出到images文件夹
                }
            }
        ]
    },
    plugins:[
        new HtmlWebpackPlugin({
            //自动复制这个html， 将css和js导入
            template:'./index.html'
        }),
        // new CleanWebpackPlugin(),
        new MiniCssExtractPlugin({
            filename:"css/index.css"   //输出到css文件夹。
        })
    ],//插件配置
    devServer:{  //热更新
        contentBase:resolve(__dirname,"build"),
        compress:true,//启动压缩
        port:3001, //端口
        open:true,//自动打开浏览器
        hot:true,
    },
    mode:"development"     //开发模式
}

    # js兼容处理

    • js的兼容处理需要使用babel和Polyfill,Babel是一个广泛使用的 ES6转码器，可以将 ES6代码转为 ES5代码,Polyfill的准确意思为，用于实现浏览器并不支持的原生API的代码。
    • js兼容性处理：需要下载 babel-loader, @babel/core。基本js兼容性处理@babel/preset-env问题：只能转换基本语法，如promise高级语法不能转换。全部js兼容性处理@babel/polyfill问题：只要解决部分兼容性问题，但是将所有兼容性代码全部引入，体积太大了。需要做兼容性处理的就做：按需加载core-js。
    
//loader
      {
        // 第三种方式：按需加载
        test: /\.js$/,
        exclude: /node_modules/,
        loader: 'babel-loader',
        options: {
          // 预设：指示babel做怎样的兼容性处理
          presets: [
            '@babel/preset-env', // 基本预设
            {
              useBuiltIns: 'usage', //按需加载
              corejs: { version: 3 }, // 指定core-js版本
              targets: { // 指定兼容到什么版本的浏览器
                chrome: '60',
                firefox: '50',
                ie: '9',
                safari: '10',
                edge: '17'
              },
            },
          ],
        },
      },
