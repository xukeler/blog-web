    # JsBridge

    • JsBridge是App混合开发中解决原生Native和前端js之间的通信的桥梁，能够让前端的js也能调用Native的方法，它的核心是构建Native 和非 Native间消息通信的通道，而且这个通信的通道是双向的。

    # JavaScript 调用 Native

    • js调用native有两种方式注入API和拦截URL SCHEME ,注入API就是通过WebView提供的接口，向 js的window对象中加入native方法，使用时直接js调用方法即可调用native。
    • URL SCHEME是一种类似于url的链接，是为了方便app直接互相调用设计的，形式和普通的url近似，主要区别是protocol和host一般是自定义的。
    • 拦截URL SCHEME的主要流程是：Web 端通过某种方式iframe.src、window.open、documention.location、href 发送URL Scheme请求，之后native拦截到请求并根据URL SCHEME（包括所带的参数）进行相关操作。
    • js调用 native推荐使用注入API 的方式。
    • 在js中调用alert、console、prompt、confirm等方法就会触发WebChromeClient的onJsAlert、onConsoleMessage、onJsPrompt、onJsConfirm方法的回调。
    //注入Api调用
//ios
window.postBridgeMessage(message);
window.webkit.messageHandlers.nativeBridge.postMessage(message);
//android
window.AnalyticsWebInterface.showErrorFunction(message);


//协议调用
//协议格式://协议名称/参数
documention.location="Js_SCHEME://GETDATA?params=data"
prompt("Js_SCHEME://GETDATA?params=data")

    # Native调用JavaScript

    • native调用js比较简单，在js写的方法会注册到window对象上，native直接调用即可。

    • JsBridge有一个插件把js和native之间的调用进行封装，需要在native端注入这个文件。
