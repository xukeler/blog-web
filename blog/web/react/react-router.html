    # 路由组件和一般组件

    • 写法不一样，一般组件<App/>,路由组件需要Route匹配，< Route path="/demo" component ={Demo}/>。
    • 接收的内容不一样，一般组件接收props，传递才能获取到值，否则就是空对象，路由组件固定接收到一个路由对象，包含history,location,match。
    
history:
    go: ƒ go(n)
    goBack: ƒ goBack()
    goForward: ƒ goForward()
    push: ƒ push(path, state)
    replace: ƒ replace(path, state)
location:
    pathname: "/about"
    search: ""
    state: undefined

match:
    params: {}
    path: "/about"
    url: "/about"

    # NavLink封装

    • Link 有一个to属性设置要跳转的路径，也可以传递一个对象，在对象里面传递参数。
    • NavLink是一个特殊的Link,也是具有一个to属性，可以传递路径或对象。可以设置当前路由高亮activeClassName，高亮路由的class。
    • 在界面通过NavLink的方式进行路由跳转，一旦NavLink的属性设置的过多，需要跳转的地方也很多，需要写很多的NavLink，这个时候就需要封装一下。
    • Route为指定路径定义匹配组件，path匹配路径，component匹配组件，还可以使用render的方式设置组件。
    • 组件除了可以通过属性props传递参数，也可以通过标签体传递内容，类似于插槽，传递了标签题，在子组件的props可以接收到children属性。
    • 标签体实际上就是在标签>内容<加入内容，也可以在标签通过children传递。
    • 在标签上定义children属性，实际就是在向标签内填入内容，内容就是children属性值。
    //父组件
import React ,{Component} from "react"
import "./index.css"
import {NavLink,Link,Route} from "react-router-dom";
import Home from '../home';
import Other from '../other'
import Nav from './Nav'
export default class Header extends Component{
    render(){
        return(
            <div>
                <header className="title">
                    <Nav title="home"  to="/home">Home</Nav>
                    <Nav title="other"  to="/other" children="other"></Nav>
                </header>
                <div className="container">
                    <Route path="/home" component={Home}></Route>
                    <Route path="/other" component={Other}></Route>
                </div>
            </div>
        )
    }
}

//封装组件
import React, { Component } from 'react'
import {NavLink} from "react-router-dom"
export default class Nav extends Component {
    render() {
        return (
            <NavLink className="action" {...this.props}></NavLink>
        )
    }
}

    # Switch

    • Switch组件的作用就是相同的路径，一旦匹配到组件，就不会向下继续匹配相同路径不同组件的Route。保持匹配路由的唯一性。

    # 严格匹配exact

    • 默认路由是模糊匹配，模糊匹配会带来一个问题就是/home/menun能匹配到/home的组件，因为开头相相同，所以模糊匹配能匹配上。
    • 严格模式就是什么路径匹配什么路由，不管路径开头相不相同,在route上添加exact属性就是严格模式。
    • 严格模式不要随便开启，出现问题时再开启。
    <Switch>
    <Route exact path="/home" component={Home}></Route>
    <Route exact path="/other" component={Other}></Route>
    <Route exact path="/other" component={Person}></Route>
</Switch>

                        <Route exact path="/person" component={Person}></Route>
                        <Redirect to="/home"/>
                    </Switch>

                </div>
            </div>
        )

    # 多级路由

    • react中路由的注册是有顺序的，因此在匹配的时候也是按照这个顺序进行的，因此会先匹配父组件中的路由,多级路由的实现依赖于模糊匹配，在路由匹配中，优先父组件的路由匹配，子路由的路径需要把父路由的路径加上，模糊匹配就会把父路由的组件显示出来，父路由匹配完成之后就会匹配子路由，然后显示匹配到子组件。
    • 不要轻易的使用精确匹配。
    //home父组件
    render(){
        return(
            <div>
                <header className="title">
                    <button onClick={this.updateSate} >修改name</button>
                    <button onClick={this.sendMsg} >兄弟通信</button>
                    <Nav title="home"  to="/home">Home</Nav>
                    <Nav title="other"  to="/other" children="other"></Nav>
                    <Nav title="person"  to="/person" children="person"></Nav>
                </header>
                <div className="container">
                    <Switch>
                        <Route  path="/home" component={Home}></Route>
                        <Route  path="/other" component={Other}></Route>
                        <Route  path="/person" component={Person}></Route>
                        <Redirect to="/home"/>
                    </Switch>

                </div>
            </div>
        )
    }


//子组件
    render() {
        return (
            <div>
                <header style={{height:"40px",width:"100%"}}>home组件</header>
                <div>
                    <Nav to="/home/list" children="list"></Nav>
                    <Nav to="/home/content" children="content"></Nav>

    # params传参

    • params在路径的后面加上:key ?问号表示参数可传可不传,多个参数用/隔开和vue的params传参一样，刷新不会消失,参数通过this.props访问到。
    • 在子组件接收到路由对象，里面的match里面的params包含传递的参数。
    //父组件
    render() {
        return (
            <div>
                <header style={{height:"40px",width:"100%"}}>home组件</header>
                <div>
                    <Nav to={`/home/list/${45454}/小明`} children="list"></Nav>
                    <Nav to="/home/content" children="content"></Nav>
                </div>
                <div>
                    <Switch>
                        <Route path="/home/list/:id?/:name" component={List}></Route>
                        <Route path="/home/content" component={Content}></Route>
                        <Redirect to="/home/list"/>
                    </Switch>
                </div>
            </div>
        )
    }
//子组件接收参数
import React, { Component } from 'react'

export default class List extends Component {
    render() {
        let {id,name} =this.props.match.params
        return (
            
            <div>
               <div>{id}</div>
               <div>{name}</div>
            </div>
        )
    }
}

    # search(query)传参

    • search实际上就是query传参，在路径后面?隔开,多个参数&符隔开，键值对的方式拼接。
    • 接收seach参数，在目标路由组件的路由对象中location对象中的search就是参数,只不过它是一个字符串，不是处理好的对象，需要手动处理,借助querystring库来转换，当然也可以手动转换，转换成一个对象就是我们想要的参数。
    //跳转
    render() {
        return (
            <div>
                <header style={{height:"40px",width:"100%"}}>home组件</header>
                <div>
                    <Nav to={`/home/list/${45454}/小明`} children="list"></Nav>
                    <Nav to="/home/content?id=435534&name=dedsfhdsf" children="content"></Nav>
                </div>
                <div>
                    <Switch>
                        <Route path="/home/list/:id?/:name" component={List}></Route>
                        <Route path="/home/content" component={Content}></Route>
                        <Redirect to="/home/list"/>
                    </Switch>
                </div>
            </div>
        )
    }


//目标路由
import qs from 'querystring'
    render() {

    # 编程式路由

    • 在页面级组件中(通过route注册过的组件)，可以通过this.props.push this.props.replace来进行跳转。
    • 普通的组件没有被注册路由的，访问props是没有history的，可以通过withRouter的方式获取到history,在暴露组件的时候将定义的类传入withRouter导出。
    • Prompt作用就是离开当前网页的提示
    import React from "react"
import {withRouter,Prompt} from 'react-router-dom'
class WithRouter123 extends React.Component{
    constructor(){
        super();
        this.state={
            val:""
        }
    }
    componentDidMount(){
        console.log(this.props)
    }
    jump=()=>{
        console.log(this.props)
        //search 
        // this.props.history.push("/meeting?id=123&name=ming")
        //params
        // this.props.history.push("/meeting/1232342/ming")
        //state
        // this.props.history.push("/meeting",{id:123,name:ming})
        this.props.history.replace("/meeting");
        // 在页面级才能访问到history 页面级指使用route注册的页面，
        // 组件里面是访问不到的
        //没有注册路由的普通组件想要实现编程式导航可以使用withRouter
        //引入withRouter ，withRouter(WithRouter123)将组件暴露出来即可
        //Prompt作用就是离开当前网页的提示  
    } 
    change=(e)=>{
        this.setState({
            val:e.target.value
        })
    }     
    render(){
        return(
            <div >
                    <Prompt when={this.state.val.length>0} message={"确定要离开吗"}/>
                    <button onClick={this.jump}>跳转meeting</button>
                    <input type="text" value={this.state.val} onChange={this.change} />
            </div>
        )
    }
}
export default withRouter(WithRouter123)

    # BrowserRouter和HashRouter的区别

    • BrowserRouter使用的是h5的history api，不兼容IE9以下版本，HashRouter使用的是URL的哈希值。
    • BrowserRouter的路径没有包含#,HashRouter路径包含HashRouter.。
    • BrowserRouter刷新后state参数还存在。HashRouter刷新后state参数刷新后消失。

    # 路由懒加载

    • react的路由懒加载需要从react引入lazy函数和Suspense。
    • route要被Suspense包裹，定义一个fallback用在没有加载出来的展示，可以显示一个loading。
    //引入
import React ,{Component,lazy,Suspense} from "react"

//引入路由
const Home=lazy(()=>import("../home"))
const Other=lazy(()=>import("../other"))
const Person=lazy(()=>import("../person"))

//route要被Suspense包裹

    <Suspense fallback={<h1>loading</h1>}>
        <Switch>
            <Route  path="/home" component={Home}></Route>
            <Route  path="/other" component={Other}></Route>
            <Route  path="/person" component={Person}></Route>
            <Redirect to="/home"/>
        </Switch>
    </Suspense>
