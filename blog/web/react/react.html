    # React框架是什么

    • React是一个将数据渲染为html用户界面的 JavaScript 库，主要用于构建用户UI界面。由facebook开发且开源。

    # React框架的特点

    • React通过对DOM的模拟，最大限度地减少与DOM的交互。JSX是 JavaScript语法的扩展。React开发不一定使用 JSX，但是建议使用它。通过 React构建组件，使得代码更加容易得到复用，能够很好的应用在大项目的开发中。React实现了单向响应的数据流，从而减少了重复代码。

    # React开发

    • 使用React开发需要引入两个js文件，一个是react.min.js React 的核心库,react-dom.min.js提供与DOM 相关的功能.
    • 通过脚手架工具create-react-app 创建的项目，在入口的index.js文件引入的react只做逻辑层的操作，react-dom做渲染层，渲染DOM，使用移动端可以换到背的渲染库。

    # react JSX语法原理

    • react是函数式编程，使用jsx语法来渲染组件。jsx语法是合法的JavaScript代码，看上去和我们平常写的html代码差不多，只不过html代码片段写在了JavaScript里面，这就是jsx语法。
    • jsx的实质就是React.createElement方法的调用,它的作用就是创建虚拟Dom。
    class Header extends Component {
  render () {
    return (
      <div>
        <h1>React 小书</h1>
      </div>
    )
  }
  //
  render () {
   return   React.createElement('div',{},React.createElement('h1',{},'React 小书'))
  }
}

ReactDOM.render(
  <Header />,

    • 在javascript中，每一个DOM元素都可以用JavaScript对象来表示，一个DOM元素一般包含标签名，属性，样式，子元素或内容，这些都可以用合法的JavaScript语法来表示。这个看起来就和vue的render函数一样了，其实vue的render函数也是借鉴了react的jsx语法。我们在写的时候是以标签的形式写的，react.js会自动编译成下面的JavaScript对象来表示，两者表示的结构和信息是一致的。

    # 虚拟Dom

    • 虚拟Dom本质就是一个对象，这个对象包含了要渲染的真实Dom的信息，例如子节点的信息，自身的属性id class，以及内容，哪个类型的标签。
    • react的diff算法会将虚拟Dom进行比较，更新差异的真实Dom，实现局部页面更新。
    • 虚拟Dom中的key是Dom对象的唯一标识，当状态的数据发生变化的时候，新数据会生成新的虚拟Dom,然后新旧虚拟Dom会进行diff比较，旧的Dom中找到和新的Dom相同的key，内容没有变化那么就复用旧的真实Dom，内容变化，就生成新的真实Dom，覆盖旧的真实Dom，旧的虚拟Dom没有找到和新的虚拟Dom相同的key，就直接生成新的真实Dom。
    • 列表渲染使用index作为key,一旦数据发生变化，index也会跟随变化，但是绑定index的key的Dom没有变化，但是两者之间的对应关系发生变化，在进行diff比较的时候，旧的真实Dom不能复用，会产生没必要的真实Dom更新，页面的更新效率降低。如果结构中还包含输入类的Dom例如input，就会造成界面显示问题。

    • 在jsx结构可以插入JavaScript表达式，需要用{ }包裹起来，表达式的结果会渲染到页面，{ }里面可以放任何的表达式，包括jsx结构。jsx也能像普通元素那样赋值，能够作为参数传递。

    # 函数式组件(简单组件)

    • 函数式组件就是在函数内部把jsx结构返回出来，把函数名写成一个标签，然后在解析的时候发现是个函数定义的组件，就调用这个函数渲染，把返回的虚拟Dom渲染成真实Dom。
    • 函数式组件内部的this是undefined，因为编译后开启了严格模式。
    • 函数式组件没有状态管理,也没有声明周期。
    • 组件名称要大写。
    function App() {
  let data=['1',"2",'3']
  return (
    <div className="App">
      <header className="App-header">
          <ul>
            {
              data.map((item,index)=>{
                return <li key={index}>{item}</li>
              })
            }
          </ul>
      </header>
    </div>
  );
}

export default App;

//渲染
ReactDOM.render(
  <App></App>,  //
  document.getElementById('root')

    # 事件绑定

    • 在react.js监听事件形式和原生事件更吻合，也是通过onClick，onChange等事件名前面加on，这点和vue不同，vue是通过@click事件名前面加@来绑定事件。
    • 绑定的事件需要写在组件内部，用{}包裹起来，通过on实现的事件绑定，只能用在普通的html标签，而不能用于组件标签。
    • 绑定的事件也会有event对象，只不过react.js将event对象进行了封装，提供统一的api
    • 在事件里面不能通过this访问到当前的实例对象，需要使用bind把实例方法绑定到当前实例上，然后在函数内部就能够访问 this实例，在调用的使用不影响传参。
    • 需要绑定this原因是因为方法并不是通过实例调用的，是直接调用，所以this为undefined。
    • 也可以使用es6的箭头函数，这样就不用bind this
    class Everything extends Component{
    bodyClick(){

    click=()=>{//箭头函数确定this指向

        return (
            <div onClick={this.bodyClick.bind(this)} className="Header" style={bodyStyle}>
              我是身体
              <button onClick={this.click}>点击</button>
          </div>
    )
}

    # 组件state状态设置

    • 每一个组件都会有一个自己的数据状态（state）,类似于vue的data，用来存储组件的数据状态。

    • 设置state状态在construstor里面设置，constructor方法是es6类的默认方法，一个类必须要有一个constructor方法，默认返回实例对象this。

    • 改变state是使用setState方法，它接受一个对象或者一个函数作为参数，作为一个对象的时候，传入更改的部分即可，因为setState是异步的，调用setState并不会立即执行，所以在改变state的函数里面打印的还是旧的值，但是新的值已经渲染到页面。

    • 函数作为setState的参数的时候，修改state依赖于旧的值就得用函数作为参数,函数接收两个参数，一个是state,一个是props。

    • state接收第二个参数为回调函数，会在render结束之后调用，获取state的更改后的值，需要在setState的回调函数里面访问到,也可以结合promise async 获取。

    # props组件传参

    • 使用props和vue的差不多，只不过react的props不需要v-bind，直接按照标签属性的格式书写就行。
    • 在接收props的组件内通过this.props.参数名来访问传进来的参数，传进来的参数在接收组件内是无法更改的，只能在传入组件更改。

    # 生命周期

    • react在组件渲染并插入到页面会有一个生命周期，和vue的生命周期类似。可以像render一样定义在组件的内部。
    • contrustor组件内部的状态初始化都是在这里面进行，最先调用。
    • componentWillMount在render函数之前调用,即将废弃避免使用。。
    • componentDidMount在render函数调用之后调用。
    • componentWillUnmount组件卸载前调用。
    • componentWillUpdate组件重新渲染时开始调用,即将废弃避免使用。。
    • componentDidUpdate组件更新完毕时调用接收三个参数，第一个旧的props，第二个旧的state,第三个getSnapshotBeforeUpdate的返回值。
    • componentWillReceiveProps props改变时调用,即将废弃避免使用。
    • shouldComponentUpdate控制组件是否向下渲染。
    • forceUpdate强制更新可以不用经过shouldComponentUpdate这一步，直接就更新，
    • getDerivedStateFromProps静态声明周期componentWillMount componentWillUpdate冲突，在调用 render 方法之前调用，并且在初始挂载及后续更新时都会被调用。它应返回一个对象来更新 state，如果返回 null 则不更新任何内容。接收传递过来的props和实例的state一旦调用，属性名相同的state便无法更改。使用较少，state的值依赖于props可以使用。
    • getSnapshotBeforeUpdate在render之后，componentDidUpdate更新之前调用，返回一个参数能被componentDidUpdate接收，能在发生更改之前从 DOM 中捕获一些信息（例如，滚动位置）。

    提示

    版本更新，生命周期函数有变动，以官方文档为准。

    # DOM操作

    • 虽然react提供了很多的方法进行事件监听，有些功能的实现避免不了DOM的操作，所以需要使用ref来操作DOM。
    • ref为回调函数的时候，函数的参数就是当前的DOM，当元素挂载完成之后，就会执行函数。
    • ref为字符串的时候可以通过this.refs.字符串访问DOM
    • 可以给html标签加ref，也可以给组件加ref(旧版本,官方文档已删除)。
    • 新的操作Dom方式通过React.createRef()来实现,React.createRef()调用后返回一个容器，里面存储被ref表示的节点，但是只能存储一个，如果要使用多个，需要多次调用React.createRef(),官方推荐使用。
    • 一般不提倡操作DOM

    提示

    当ref为内联函数的时候，在更新过程中会触发两次，一次是null，一次是DOM，不影响使用。

    旧版本

    render(){
        return(
            <div className="list-wrapper">
                <div dangerouslySetInnerHTML={{__html: this.state.html}}></div>

    # constructor 构造器

    • 在类组件中，发现constructor写与不写没有什么区别，官方文档说明constructor在new实例调用初始化state，给事件绑定this,但是state在constructor外面也能初始化，事件的this指向可以使用箭头函数解决。
    • constructor中的props传与不传有些区别，不传在外面也能访问到this.props，但是在constructor内部不能通过this.props访问props，必须通过super(props)传递,才能在constructor访问this.props。
    • 在开发中可以省略。
    
class MenuData extends Component{
  state{
    name:"MING"
  }
  constructor(props){
    super(props) //必须传递
    console.log(this.props)
  }
  render(){
    return(
      <div>props</div>
    )
  }
}

    # 高阶函数和函数柯里化

    • 一个函数接收一个函数参数，那这个函数就是高阶参数，一个函数返回一个函数，那这个函数也是高阶参数。常见的高阶函数forEach setTimeut map some every filter promise 等。
    • 函数柯里化指的是通过函数调用继续返回函数，继续调用，实现多次接收参数，最后统一处理就是函数柯里化。
    • 在处理多个input表单请求，需要获取value，如果每个单独绑定onChange事件，需要写很多，解决这个问题用到高阶函数，以及柯里化，react绑定事件不能加()否则视为函数调用，但是解决这个问题需要加()来传递input的类型，所以要返回一个函数获取 event对象拿到value。
    //示例多个表单请求

import React,{Component} from "react";

export default class HeightFun extends Component{
    state={
        name:"",
        phone:'',
        code:''
    }
    inputChange=(type)=>{
        return (event)=>{
            this.setState({
                [type]:event.target.value
            })
        }
    }
    submit=()=>{
        console.log(this.state)
    }
    render(){
        return(
            <div>
                <header>
                    高阶函数
                </header>
                <section>
                    <p>
                        姓名：<input onChange={this.inputChange("name")} type="text"/>
                    </p>
                    <p>
                        手机号：<input onChange={this.inputChange("phone")} type="text"/>
                    </p>
                    <p>
                        验证码：<input onChange={this.inputChange("code")} type="text"/>
                    </p>
                    {/* 还可以加更多表单，调用同一个事件，通过传参判断 */}
                    <p>
                        <button onClick={this.submit}>提交</button>
                    </p>
                </section>
            </div>
        )
    }
}

    # 发布订阅通信

    • 兄弟组件通信通过pubsub.js，依靠于消息订阅和发布机制。订阅和发布机制简单说有一个事件调度中心，有人会订阅消息，因为有人订阅消息，就会有人发布消息，一旦订阅就会给订阅发送消息。
    • 需要下载yarn add pubsub-js，在组件内引用。
    • subscribe订阅消息，回调函数接收两个参数，第一个是约定的消息名称，第二个是传过来的参数。
    • 取消订阅unsubscribe。

    TIP

    组件销毁要取消订阅

    • 文档地址
    import Pubsub from 'pubsub-js';
//订阅消息
       let data= Pubsub.subscribe('brother',(type,data)=>{
            console.log(type,data)
            this.setState({
                name:data.name,
                des:data.des
            })
        })
      Pubsub.unsubscribe(data)  //取消订阅
//发布消息
 Pubsub.publish("brother",{name:"chat",des:"我是你兄弟"})

    # PureComponent

    • 类组件继承的是react的Component，但是会有一个问题，父组件中的状态改变，但是不牵扯到子组件，子组件也会重新调用render渲染，这个渲染没有必要，子组件什么都没有更新。所以为了解决这个问题，从 PureComponent继承。
    import React, { Component,PureComponent } from 'react'
import {withRouter} from "react-router-dom"
class Footer extends PureComponent {
    render() {
        return (
            <div>
                <div>footer</div>
            </div>
        )
    }
}
export default withRouter(Footer)
