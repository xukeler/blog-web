    # es6

    • 是 JavaScript 语言的下一代标准,2015年发布，上一代是es5,每一次的迭代都会新增一些语法,es6新增的promise,async await ,proxy等新的语法，能够解决开发中一些问题，提高开发效率。浏览器对新的语法支持可能没有支持，需要进行转义，将es6转义es5。

    # let和const

    • 在之前声明变量通过var，var声明变量会带来变量提升的问题，而let 和const则没有，可以重复声明。
    • let 声明变量存在块级作用域，
    • const声明变量存在块级作用域，声明必须赋值同时进行，只能声明常量，一旦声明不能改变，不可以重复声明。
    
console.log(count) //undefined  声明没有赋值，声明提前
var count=1;
var count=345;  //可以重复声明
console.log(count) 

for(var i=0;i<10;i++){
    console.log(i)
}
console.log(i)  //10 i是一个全局变量。


for(let i=0;i<10;i++){
    console.log(i)
}
console.log(i)  //报错  i is not defined  存在块级作用域

    # 展开运算符

    • 展开运算符是用三个...表示，一般用于展开数组合并数组，或者展开对象，克隆对象实现浅拷贝，合并对象，也可用于获取函数的参数，当参数很多的时候。
    • 展开对象的时候必须用{}包裹，合并对象，属性相同时，会覆盖属性值。
      let arr=[1,2,3,4,5];
  let arr1=[6,7,8];
  let arr2=[...arr,...arr1] //合并数组
  console.log(...arr,arr2) //展开数组

  let obj={
      name:"123"
  }
  console.log({...obj}) //展开对象
  console.log({...obj,count:567,name:"ming"}) //{name: "ming", count: 567}  合并对象

    # 箭头函数

    • ES6 允许使用“箭头=>定义函数。箭头函数和传统的函数使用时会有一定区别，箭头函数简化了传统函数的写法，在代码量上减少了很多，也更直观。但是使用时有一些注意事项。
    • 箭头函数没有自己 this,函数体内的this对象，就是定义时所在的对象，而不是使用时所在的对象。和直接写function的this指向不同。
    • 箭头函数的 this 看外层的是否有函数，如果有，外层函数的 this 就是内部箭头函数的 this，如果没有，则 this 是 window。
    • 不可以当作构造函数，也就是说，不可以使用new命令，否则会抛出一个错误。
    • 不可以使用arguments对象，该对象在函数体内不存在。function是可以使用的。
    let res=function (){
alert("123")
}
等同于
let res=()=>{
alert("123")
}

var f = () => 5;
// 等同于
var f = function () { return 5 };

//没有参数
let fun1 = () => console.log('我是箭头函数')
fun1()
// 只有一个参数的时候，()可以省略
let fun2 = a => console.log(a)
fun2('aaa')
// 两个及两个以上参数的时候，()不能省略
let fun3 = (x, y) => console.log(x, y)
fun3(25, 35)
// 函数体的情况
// 函数体只有一条语句或表达式的时候，{}可以省略-->会自动返回（return）语句/表达式的结果
fun4 = (x, y) => x + y
console.log(fun4(24, 36))

// 函数体不止一条语句或表达式的时候，{}不可以省略
let fun5 = (x, y) => {
  console.log(x, y)
  return x + y
}
console.log(fun5(20, 30))

    # promise

    • Promise是异步编程的一种解决方案,它相当于一个容器，里面保存着未来才会结束的事件，它也主要是用来解决回调地狱的问题。
    • promise是一个对象，代表一个异步操作，有三种状态，进行中，成功，失败。只有异步操作的结果的可以决定当前是哪种状态，promise一旦新建执行，就没有办法中途停止。
    • promise允许函数作为参数传入，res为成功的回调，rej为失败的回调，当条件满足时，res/rej会将成功或失败的结果暴露出来。
    • 通过.then拿到暴露出来的结果，.then是一个全新的promise函数，.then允许两个函数作为参数，一个函数拿到成功的暴露结果，一个拿失败暴露的结果。
    • 通过catch获取一个指定发生错误时的回调函数。error为错误信息。
    • 一般来说，不要在then方法里面定义 rej状态的回调函数（即then的第二个参数），总是使用catch方法。
    const pro=new Promise((res,rej)=>{
        if(!true){
          res('123')
        }
        else{
          rej("4576")
        }
      }).then((res)=>{
        console.log(res)
      },(rej)=>{
        console.log(rej)
      }).catch((error)=>{
    console.log(error)
}

    # export和export default

    • ES6模块主要有两个功能：export和import。export和export default都是用于对外输出本模块，均可用于导出常量、函数、文件、模块等。
    • export和export default输出的模块需要使用import导入才能使用。
    • 在一个文件或模块中，export、import可以有多个，export default仅有一个。
    • 通过export方式导出，在导入时要加{ }，export default则不需要。
    import postUrlEnd from './url';
import {
    get,

    ###async await

    • es6新增的sync是用来解决异步编程的函数，最常见的异步编程就ajax请求，一般开发中都会遇到一个场景，就是封装了一个ajax请求函数，需要获取请求的结果才能进行下一步的操作，那这时候，就不能直接用变量接收ajax的返回值，因为不知道什么时候请求完成，而js的代码运行是不会等待请求的完成，它一直向下运行，这个时候变量接收的值是undefind，只能把逻辑操作放到请求成功的回调中。es6里面解决这个问题推出了sync函数。
    • 使用sync函数，需要在函数前面加上sync标识，表示这是一个异步函数，使用await命令等待异步请求的结果（需要在请求成功的回调中return一个值），用变量接收,await运行完毕之后才会继续向下运行，sync函数返回的是一个promise，所以也可通过.then的方式拿到异步结果。
    //普通函数
function set(){
            var result;
            $.ajax({
                url:"https://v1.itooi.cn/netease/banner",
                type:"GET",
                success:function(res){
                    result=res;
                    return res
                }
            })
            console.log(result)//undefined
        }
        set()

// sync函数

 async function get(){
           var result=await $.ajax({
                url:"https://v1.itooi.cn/netease/banner",
                type:"GET",
                success:function(res){
                    return res
                }
            })
            console.log(result)//data
            return result;
        }
get().then((res)=>{
            console.log(res)//data
        });

    # class类

    +ES5 生成实例对象的传统方法是通过构造函数。ES6 提供了更接近传统语言的写法，引入了 Class类这个概念，作为对象的模板。通过class关键字，可以定义类。

    • 定义个每个类下面都有方法和属性，访问这些方法和属性必须通过new来实现，static 定义静态属性和方法得关键字，静态属性和方法可以直接通过类名访问不需要经过new,但一般这样用得很少。
    • public定义属性公开的，可以被修改，默认值就是public。
    • private私有属性，定义属性是私有的，只能在类的内部访问。
    • readonly设置属性只读不能被修改。
    • constructor 是类得构造函数，在new实例化对象的时候就在调用构造函数constructor，在里面可以通过this(指向当前的实例对象)修改属性。通过new实例化对象的时候传递的参数，在constructor也可以访问到，方法里的this指向调用方法的对象。
    • es6通过extends 关键字来实现继承功能，继承的子类会获取到父类的方法和属性，如果有同名的，会优先选择子类的方法或属性。
    • 在子类使用constructor需要使用super()将父类的constructor的参数传递过去，必须调用super，在子类的方法里super就代表父类。
    • abstract创建一个抽象类只能用来被继承，不能实例化。
    class People{
    name="123";
    static readonly age=18;  //设置只读
    setName(name:string):string{
       return this.name=name
    }
}
let per=new People();
console.log(per.name,People.age)  //123 18
People.age=20 //报错
console.log(per.setName('ming')) // ming
