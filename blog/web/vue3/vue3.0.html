# vue3.0

  • vue3在2021年已经推出稳定版本，在2022年作为默认版本，官方文档默认是3.0，脚手架默认安装3.0版本等。
  • vue3相对于vue2来说有了很大的改动，最主要的就是composition api组合式api，作为打工人，要时刻跟上大佬的脚后跟。
  • 官网

# setup

  • vue3新增组合式api的概念和setup回调函数。组合式api使用必须调用setup方法也是组合式 API 的入口点，后期增加语法糖，就可以换种形式使用。
  • setup特点
    • setup也可以看做是组件生命周期的一部分，调用时机在beforeCreate,这个时候组件还没有创建，还访问不到this，也就不能通过this访问data computed methods。在所有的·composition API·相关回调函数中也都不可以通过this访问。
    • setup必须要有返回值给模板使用，return一个对象， 为模板提供数据, 也就是模板中可以直接使用此对象中的所有属性/方法，回对象中的属性会与data函数返回对象的属性合并成为组件对象的属性，返回对象中的方法会与methods中的方法合并成功组件对象的方法,如果有重名, setup优先。
    • setup不能是一个async函数: 因为返回值不再是return的对象, 而是promise, 模板看不到return对象中的属性数据
    • setup接收两个参数 props 和 context。
      • props是父级传给自己的props和原来一样。
      • context是一个对象，包含attr(获取当前组件标签上的所有属性的对象，等同于this.$attrs)，emit(分发事件,等同于this.$emit),slots(插槽对象，等同于this.$slots)，正常来说context对象不怎么会用到，这些方法都有别的方式实现。

注意

虽然vue3兼容vue2 的option api，但是组合式api不要和vue2 的option api混着用,混着用也就失去升级的意义了

# ref reactive响应式数据

  • 在vue2中定义一个响应式数据就直接在data中定义就可以，修改视图也会跟随改变。vue3实现响应式数据通过ref(reference对象)和reactive来实现。
  • ref定义单个响应式数据，例如，数字，字符串等基本数据类型，也可以定义对象数组复杂数据类型。
    • 使用先从vue引入ref,ref是个函数，传入初始值，返回一个响应式可改变的对象，提供了一个内部访问属性value。
    • 修改要操作.value,在模板中使用不用带.value。
    • ref在定义基本数据类型响应式依赖Object.defineProperty( )的get( )和set( )。定义复杂数据类型底层ref会借助reactive的proxy 定义响应式。也就是ref能实现reactive的功能，但reactive不能全部实现ref的功能。
  • reactive定义多个响应式数据(对象数组)且无法定义基本数据类型(number string...)，把参数加工成一个代理对象，全称为proxy对象,基于Es6的Proxy实现。
    • 使用和ref类似，唯一的使用区别是没有.value。
<template>
    <div>
        <el-button type="primary" @click="add">加</el-button>
        <el-button type="success">{{ num }}</el-button>
        <el-button type="success">{{ count }}</el-button>
        <el-button type="primary" @click="reduce">减</el-button>
    </div>
</template>

<script>
import {ref,reactive} from 'vue'
export default {
    setup () {
        let  count=ref(0); //返回响应式代理对象
        let num=reactive({        
            name:1,
            age:{
                age:5
            }
        })
        console.log(num,count)
        let add = () => {
            count.value+=1 ;
            num.age.age+=1
        };
        let reduce = () => {
            count.value-=1;
            num.age.age-=1
        };
        return {
            num,
            count,
            add,
            reduce,
        };
    },
}
</script>

# ref和reactive的区别

  • 1.ref在修改数据必须有.value,而reactive不需要，直接使用即可。
  • 2，ref可以操作所有的数据类型。而reactive只能操作复杂数据类型对象和数组
  • 3，ref和reactive都可以定义复杂数据的响应式，但是ref可以一次性修改数据，reactive不可以，reactive一次性修改数据后将会丢失响应式。
<script lang="ts">
import { reactive ,ref} from "vue";
export default {
  setup() {
    let num = reactive({
        name:1
    });
    console.log("num",num) //Proxy {name:1}
    let count=ref({
        age:18
    })
    count.value={age:89}
     num={name:89}
    console.log("num",num,"count",count) //{name:1}已经丢失响应式
    return {
      num,
        count
    };
  },
};
</script>

# reactive 定义响应式对象删除增加属性

  • 初始先定义一个正常的obj对象，在使用reactive定义为一个响应式对象，这个时候修改属性或删除增加属性，只能通过reactive(obj)返回的代理对象处理，通过obj对象修改不生效，界面不会更新。
  • 增加删除属性在ts中会出现红线提示数据类型有问题，所以要设置obj为any类型，或reactive设置any泛型。
<template>
  <div class="container">
    <div class="title">reactive增加删除属性</div>
    <div>name:{{user.name}}</div>
    <div>age:{{user.age}}</div>
    <div>sex:{{user.sex}}</div>
    <el-button type="primary" @click="del">删除</el-button>
    <el-button type="primary" @click="add">添加</el-button>
  </div>
</template>

<script lang="ts">
import { reactive} from "vue";
export default {
  setup() {
    //   let obj:any={  //修改不生效
    //     name:"小明",
    //     age:19  
    //   }
    // let user = reactive(obj);
    // const del=()=>{
    //    delete obj.name
    // }
    // const add=()=>{
    //     obj.sex="女"
    // }

    let obj={  //修改生效
        name:"小明",
        age:19  
    }
    let user = reactive<any>(obj);
    const del=()=>{
       delete user.name
    }
    const add=()=>{
        user.sex="女"
    }
    return {
      user,
      del,
      add
    };
  },
};
</script>

# 计算属性computed

  • 在setup怎么使用计算属性？使用也比较简单引入import {computed} from vue，后面使用一个api基本都是这种引入的方式。
  • computed是一个函数，需要再传入一个回调函数作为参数，如果只是get必须return 一个值。
  • 如果需要set则需要传入一个对象,对象包含get和set方法。
<template>
  <div class="container">
    <div class="title">计算属性</div>
    <input type="text" v-model="user.name">
    <div>name:{{newName}}</div>
    <input type="text" v-model="newName">
  </div>
</template>

<script lang="ts">
import { reactive,computed} from "vue";
export default {
  setup() {
      let obj={
        name:"小明",
        age:19  
      }
    let user = reactive<any>(obj);
    // const newName=computed(()=>{
    //   return user.name+1
    // })
    const newName=computed({
      get(){
        return user.name+1
      },
      set(val){
        user.name+=val
      }
    })
    return {
      user,
      newName
    };
  },
};
</script>

# 监听 watch watchEffect

  • vue3实现监听有两种方式，一种是watch，一种是watchEffect都需要引入才能使用。
  • watch接收三个参数(和旧的功能一致)
    • 监听的属性
    • 回调函数返回最新值和修改之前的值
    • 配置对象，配置是否深度监听和起始执行一次。
  • watch可以监听多个数据，使用数组来指定,如果是ref对象, 直接指定，如果是reactive对象中的属性, 必须通过函数来指定。
  • watchEffect只有一个回调函数，监听回调函数内的，用数据类型的所有属性，不需要具体到某个属性，一旦运行就会立即监听，组件卸载的时候会停止监听。
    • watchEffect默认可以深度监听，起始执行一次。
    • watchEffect可以监听多个响应式数据
  • watch 与 watchEffect 区别
    • watch可以通过配置实现watchEffect功能。
    • watchEffect只能访问当前最新的值，访问不到修改之前的值。一旦运行就会立即执行。使用时不需要具体指定监听的谁，回调函数内直接使用就可以。相比watch比较难理解。
<template>
  <div class="container">
    <div class="title">监听</div>
        <input type="text" v-model="msg.name.age.sex">
        <input type="text" v-model="val">
        <div>{{text}}</div>
        <hr>
        <input type="text" v-model="user.name">
        <input type="text" v-model="user.age">
        <div>{{group}}</div>
  </div>
</template>

<script lang="ts">
    import {defineComponent,watch,watchEffect,ref,reactive} from "vue"
    export default defineComponent({
        setup(){
            let msg=ref({
                name:{
                    age:{
                        sex:1
                    }
                }
            });
            let val=ref("");
            let text=ref("")
            //watch 监听
            // watch(msg,(value)=>{
            //     val.value=msg.value.name.age.sex+"123"
            // },{
            //     immediate:true,
            //     deep:true
            // })
            //watchEffect监听
            watchEffect(()=>{
                text.value=msg.value.name.age.sex+val.value
            })
            let user=reactive({
                name:"123",
                age:18
            })
            let group=ref("")
            //watch 监听多个数据
            watch([()=>user.name,()=>user.age],(value)=>{
                console.log(1233)
                group.value=user.name+user.age
            },{immediate:true})
            return{
                msg,
                val,
                text,
                group,
                user
            }
        }
    })
</script>

# 生命周期

  • vue3的生命周期取消了beforeCreate created这两个等同于setup,使用生命周期都需要引入，除了beforeDestroy destroyed名称完全修改，其它名称前面加个on，功能都是一样的。
    • beforeCreate created等同于setup
    • beforeMount等同于onBeforeMount
    • mounted等同于onMounted
    • beforeUpdate等同于onBeforeUpdate
    • updated等同于onUpdated
    • beforeDestroy等同于onBeforeUnmount
    • destroyed等同于onUnmounted
  • vue3兼容vue2的生命周期的写法，但是不建议使用。
<template>
  <div class="container">
    <div class="title">生命周期</div>
  </div>
</template>
<script lang="ts">
import { ref,onMounted,onBeforeMount,onUpdated,onBeforeUpdate,onBeforeUnmount,onUnmounted} from "vue";
export default {
  setup() {
    const name=ref("");
    onBeforeMount(()=>{
        console.log("onBeforeMount")
    })
    onMounted(()=>{
        console.log("onMounted")
    })
    onBeforeUpdate(()=>{
        console.log("onBeforeUpdate")
    })
    onUpdated(()=>{
        console.log("onUpdated")
    })
    onBeforeUnmount(()=>{
        console.log("onBeforeUnmount")
    })
    onUnmounted(()=>{
        console.log("onUnmounted")
    })
    return {
        name
    };
  },
};
</script>

# toRefs 响应式对象转换

  • toRefs的作用是将reactive定义的响应式对象转换为普通对象，普通对象的每个属性为ref响应式对象，相当于化整为零。
<template>
  <div class="container">
    <div class="title">toRefs</div>
    <div>{{ name }}</div>
    <div>{{ age }}</div>
    <div><button @click="change">修改</button></div>
  </div>
</template>
<script lang="ts">
import { reactive, toRefs,defineComponent } from "vue";
export default defineComponent({
  setup() {
    const state = reactive({
      name: "小明",
      age: 18,
    });
    const { name, age } = toRefs(state);
    console.log(name, age);
    const change = () => {
      name.value = "Hello";
      age.value = 89;
    };
    return {
      name,
      age,
      change,
    };
  },
});
</script>

# ref获取dom

  • vue3获取dom和vue2有些不一样，和定义响应式变量共用一个方法ref,依赖HTMLElement类型创建，默认值为null。
  • 在标签内定义的ref名称和return的名称要一至，.value就是获取的dom节点。
<template>
  <div class="container">
    <div class="title">ref获取dom</div>
    <input type="text" ref="inputElement">
  </div>
</template>
<script lang="ts">
import { ref,defineComponent,onMounted } from "vue";
export default defineComponent({
  setup() {
      let inputElement=ref<HTMLElement|null>(null);
      onMounted(()=>{
          console.log(inputElement);
          inputElement.value&&inputElement.value.focus()
      })
      return{
          inputElement
      }
  },
});
</script>

# shallowReactive 与 shallowRef

  • shallowReactive只处理了对象内最外层属性的响应式(也就是浅响应式)。
  • shallowRef 只处理了value的响应式, 不进行对象的reactive处理。
  • 和ref reactive区别
    • ref reactive是深度响应式。

# readonly 与 shallowReadonly

  • readonly深度只读数据,获取一个对象 (响应式或纯对象) ,只读取不能被修改，修改就会提示。
  • shallowReadonly浅只读数据，获取一个对象 (响应式或纯对象) ,只有第一层只能读取不能被修改，深层可以被修改。

# toRaw 与 markRaw

  • toRaw返回由 reactive 或 readonly 方法转换成响应式代理的普通对象,就是获取响应式对象的普通对象，可用于临时读取，访问不会被代理/跟踪，写入时也不会触发界面更新,和reactive功能相反。
    • toRaw不能转换ref定义的响应式变量。
  • markRaw标记一个对象，使其永远不会转换为代理，返回对象本身比如页面一些写死的数据，只是初次渲染用到，但是不会再改了，渲染具有不可变数据源的大列表时，跳过代理转换可以提高性能。

注意

markRaw使用，数据会立刻更新一次，延迟修改就不会更新

<template>
  <div class="container">
    <div class="title">toRaw-markRaw</div>
    <div>obj---{{ obj }}</div>
    <!-- <div>toRawObj---{{ toRawObj }}</div> -->
    <div>
      <button @click="change">修改数据</button>
    </div>
  </div>
</template>
<script lang="ts">
import { defineComponent, ref, toRaw, reactive, markRaw } from "vue";
export default defineComponent({
  setup() {
    const obj = reactive<any>({
      name: "hello",
      age: 18,
      list: [],
    });
    // const toRawObj=toRaw(obj); //数据修改页面没有更新
    const change = () => {
      const likes = ["a", "b"];
      // state.likes = likes
      obj.likes = markRaw(likes); // likes数组就不再是响应式的了
      setTimeout(() => {
        obj.likes[0] += "--";//数据修改页面没有更新
      }, 1000);
    };
    return {
      obj,
      // toRawObj,
      change,
    };
  },
});
</script>

# readonly 设置对象为只读

  • readonly接收由ref|reactive定义的响应式对象或普通对象，返回一个只读对象，修改对象的值就会触发警告。
<template>
        <div class="readonly-container">
            <h3>readonly</h3>
            <button @click="change">修改</button>
        </div>
</template>
    
<script setup lang='ts'>
    import {ref ,readonly,reactive} from "vue";
    const obj={
        class:1,
        sex:"girl"
    }
    const state=reactive({
        name:"james",
        age:18
    })
    // const state=ref({
    //     name:"james",
    //     age:18
    // })
    const readonlyState=readonly(state);
    const readonlyObj=readonly(obj)
    const change=()=>{
        readonlyState.age+=1 //触发警告，提示属性只读
        // readonlyState.value.age+=1
        readonlyObj.class+=1
    }
</script>

# isProxy，isReactive，isReadonly,isRef,unRef

  • isProxy检查对象是否是由 reactive 或 readonly 创建的 proxy.
  • isReactive检查对象是否是由 reactive 创建的响应式代理数据。
  • isReadonly检查对象是否是由 readonly 创建的只读对象。
  • isRef检查对象是否是由 ref 创建的响应式代理数据。
  • unRef如果参数是一个 ref定义的响应式数据，则返回内部值也就是.value，否则返回参数本身。这是 val = isRef(val) ? val.value : val的语法糖函数。
<template>
  <div class="readonly-container">
    <h3>isProxy-isReactive-isReadonly-isRef</h3>
    <button @click="change(state)">change</button>
  </div>
</template>

<script setup lang="ts">
import { ref, readonly, reactive ,isProxy,isReactive,isReadonly,isRef,unref} from "vue";
const obj = ref({
  class: 1,
  sex: "girl",
});
const state = reactive({
  name: "james",
  age: 18,
});
const readonlyObj=readonly({
    age:18
})
console.log(isProxy(obj),isProxy(state)) //false true
console.log(isReactive(state),isReactive(obj)) //  true false
console.log(isRef(state),isRef(obj))    //false true
console.log(isReadonly(state),isReadonly(readonlyObj))  //false true
function change(x:any,){
    console.log(unref(x))
}
</script>

# defineProps,defineExpose,defineEmits

  • defineProps是子组件获取父组件传递过来的props，接收一个数组和对象，配置方法没变。
  • defineEmits是子组件获取父组件传递的emit事件,使用传承没变。
  • defineExpose是子组件向父组件暴露方法或属性，在2.0版本通过this.$refs.名称就可以拿到组件的所有方法和属性，现在是通过子组件向外暴露，父组件才能通过ref拿到。
<!-- 子组件 -->
<template>
  <div class="container">
    <div class="title">emit props测试</div>
    <div>props:{{props.state}}</div>
    <button @click="handle">触发emit</button>
  </div>
</template>

<script setup lang="ts">
import { toRaw, ref } from "vue";
const props = defineProps({
  state: {
    type: Number,
    default: 5,
  },
  value: {
    type: Boolean,
    default: false,
  },
});
const a = 1;
const b = ref(2);
defineExpose({
  a,
  b,
});
const emit = defineEmits(["change", "getNumber"]);
const handle=()=>{
  emit("getNumber")
  }
</script>

# Provide / Inject

  • Provide / Inject这个在vue2用到的也比较多用于祖孙通信，vue3只是换种形式使用，使用先import，导入的是provide inject函数，每次调用只能传递一个变量或函数，传多个就需要写多个。
<!-- 父组件 -->
<template>
        <div class="readonly-container">
            <h3>provide-inject</h3>
            <div>count:{{count}}</div>
        </div>
        <div>
            <Provide></Provide>
        </div>
</template>
    
<script setup lang='ts'>
    import {provide,ref} from "vue";
    import Provide from "../../components/provide.vue"
    const count=ref(1)
    const add=()=>{
        count.value++
    }
    const reduce=()=>{
        count.value--
    }
    provide("count",count)
    provide("add",add)
    provide("reduce",reduce)
</script>

# 异步组件

  • 异步加载组件就是在定义组件的时候什么都不做，只有当组件第一次使用的时候才进行加载和渲染，正常情况下是在router中引入组件时配置，有利于优化加载速度。
  • 在2.x版本的异步组件是通过将组件定义为返回 Promise 的函数来创建的,3.0版本由于函数式组件被定义为纯函数，因此异步组件的定义需要通过将其包裹在新的 defineAsyncComponent 方法中来定义异步加载。
  • 3.0的高阶选项组件语法将component修改为loader。
import { defineAsyncComponent } from "vue";
//2.x
const SetUp = () => import( '../components/setup.vue')
//或
const SetUp = {
  component: () => import('../components/setup.vue'),
  delay: 200,
  timeout: 3000,
  error: ErrorComponent,
  loading: LoadingComponent
}
//3.0

const SetUp = defineAsyncComponent(() => import('../components/setup.vue'))
//高阶
const SetUp = defineAsyncComponent({
  loader: () => import('../components/setup.vue'),
  delay: 200,
  timeout: 3000,
  error: ErrorComponent,
  loadingComponent: LoadingComponent
})

# 移除过滤器

  • 过滤器filters在2.x版本用来处理通用文本格式，和计算属性差不多用的也不多，可以用计算属性代替，3.0移除。

# 应用api

  • 改变全局Vue行为的 API 现在被移动到了由新的createApp方法所创建的应用实例上。
  • component注册全局组件和旧版本一样
  • directive注册全局自定义指令和旧版本一样
  • mixin全局混入，这个在vue2会用到的比较多，vue3推荐使用hooks。
  • use安装插件，这个用的比较多，经常见到的app.use(router)等。
