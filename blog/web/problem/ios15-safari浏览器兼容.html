    # ios15-safar浏览器兼容

    • ios升级到15之后，原本正常的页面，出现了兼容性的问题，问题的根本原因是ios15 的safari调整了导航栏，增加了一些功能，例如网站着色，这些都会影响到现有的产品交互。每次手机的系统的升级都会带来一些问题。

    # 单页面网站着色问题

    • 如果你的网站的有的页面的背景色不是白色，而是蓝色红色这类的深色背景 ，路由跳转到白色背景的页面，页面就会被着色，变成深色背景。
    • 这个着色可以在设置->safrai里面关掉。
    • 虽然可以手动关掉，但是不可能让每个用户手动操作关闭设置，因为是最近升级的网上也没有什么类似的案例，所以自己测试了一些解决方案，最终找到了解决方式，在safari浏览器单页面(nuxt ssr)的深色背景向浅色背景的跳转由this.$router.push改为location.href，即可解决问题。

    # 底部导航栏伸缩兼容

    • 原来safari的导航栏在顶部，现在升级调整到底部，也可以通过设置改回来，还是那个问题， 不可能让所有的用户来修改，还是得有一套比较合适的解决方案。
    • 有的页面底部有固定定位，当页面滚动时，底部的导航栏没有跟随手机的上移下移进行伸缩展开，那这个页面的滚动就是有问题的。当在其它页面底部导航栏收缩之后，跳到有底部固定定位的页面时，导航栏也会收缩，但是页面会空出一部分。
    • 产生这个问题的原因是滚动的容器设置了overflow-y:auto,去掉这个设置，让内容撑开容器产生滚动条就正常。
    • 虽然问题解决了，但是对于后面的开发也会多出一些顾虑，如果有页面只有局部内容滚动，且占满大半个屏幕，也会产生这个问题，使用第三方滚动插件同理也会有问题。
