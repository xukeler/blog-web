    # harmony鸿蒙

    • 鸿蒙是华为推出的一个操作系统，用来替代安卓，属于一个新的操作系统。然后他们提出一个分布式的概念，大概意思就是一个操作系统不管是手机手表，还是汽车都可以用，那对于开发来说是不是可以一套代码应用多个平台呢？
    • 作为前端开发人员对这个手机操作系统没有太多的关注，需要关注的是鸿蒙应用的开发，毕竟这也是未来一个开发走向，作为开发人员，时刻对技术保持敏感。
    • 官网

    # harmonyOS应用开发

    • 最近鸿蒙的发布会也快到了，我们公司也是和鸿蒙有一些合作需要将我们的产品开发到鸿蒙上，但是由于鸿蒙的生态不够健全，导致我们需要重新开发一套代码，不能用已有的安卓App直接转换为鸿蒙应用，不得已开发也延期了。
    • 研究了鸿蒙的开发文档整个开发方式有以下几种,其中c/c++是用于设备开发，其它是引用开发。
      • c/c++开发，这个就属于底层的开发了，不适用于大多数人，适合大佬们玩一玩。
      • java开发，和鸿蒙的技术人员沟通过，他们不推java开发，主要是维护，如果之前是安卓的开发者，应该非常容易上手。
      • js开发，纯js开发受众比较广，门槛低，但同时也有一个问题，就是鸿蒙提供的一些系统能力，js的api是没有的，只能开发简单的应用。
      • js+java混合开发，这也是鸿蒙主推的开发方式，js开发前端能够提供优美的界面，java提供强大的后台能力。
    • 开发平台除了手机端，还有电视，汽车车机，手表，平板，耳机等。
    • 鸿蒙也提供了自己的开发者工具DevEco Studio,作为前端开发人员用惯了vscode,真用不惯这个玩意，而且不支持汉化。

    # js开发鸿蒙应用

    • 作为前端开发人员关注的当然是js怎么开发一个鸿蒙应用，鸿蒙系统对js开发支持度有多高，但是文档研究下来发现，对于js的支持能力真的不够多，还不如微信小程序提供的能力多，举个例子，小程序里面的tabBar,自定义导航栏，拖动，webview这里统统不支持，就连原生的导航栏的样式，js都控制不了，和他们的技术人员沟通了，他们现在还在一个beta阶段，后续会收集开发者们的建议来弥补这部分能力，但是具体是什么时候就不得而知了。
    • 为什么拿js开发鸿蒙应用和小程序比呢，因为它们的语法结构类似，项目结构类似，都是html+ css+ js三个文件组成，只不过鸿蒙缺少一个json文件配置。
    • 会小程序开发或vue开发很快就会上手，但是要开发一个完美的应用,以鸿蒙现在的js能力是不够的，就需要借助java。

    # 总结

    • 总结下来就是现阶段不知道鸿蒙os会发展到哪个地步，有可能是未来前端发展的一个趋势，现在前端能做的东西越来越多，笑到最后的还是js，所以为了未来职业的发展方向，还是有必要学习一下，说不定就是下个技术风口。
