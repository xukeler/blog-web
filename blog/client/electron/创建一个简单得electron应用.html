    # 创建一个简单得electron应用

    • electron是一个可以用JavaScript和html css开发桌面应用得框架，例如window端exe程序，mac端得pkg dmg程序，还有Linux端程序。

    • 使用electron要安装node在应用程序中可以使用node api，和原生得window api。

    • electron包含了三个部分，一个是node，一个是浏览器渲染界面，一个是原生内容。

    • electron包含两个进程，一个主进程，一个渲染进程，主进程管理所有得渲染进程，渲染进程只管理界面得渲染。

    • # 创建一个初始electron 应用

    • electron得创建可以按照官方文档方式创建，也可以使用electron-vue来创建。

    • electron-vue是一个将electron和vue结合起来得开发框架，能使用vue的代码结构开发应用。

    • electron-vue文档地址

    # 开始搭建

    • 下载vue-cli和一个electron模板,在它的基础上开发，下载下来的模板已经配置好，只有一个界面，二次开发就行。
    • 注意不要勾选eslint，很烦人的这个东西,已经配置了也不要怕，到配置文件中删了就行。
    • 在执行vue inti过程会遇到问题无法加载 C:\Program Files\node.js\vue.psl，需要在命令行设置
    set-executionpolicy remotesigned

    # 启动程序

    • 启动程序npm run dev，启动成功之后就是这样。

    WARNING

    但是启动的时候遇到一个问题

    • 解决方案
    • 在.electron-vue目录下的webpack.renderer.config.js文件中，找到new HtmlWebpackPlugin开头的代码,替换成
    new HtmlWebpackPlugin({
      filename: 'index.html',
      template: path.resolve(__dirname, '../src/index.ejs'),
      minify: {
        collapseWhitespace: true,
        removeAttributeQuotes: true,
        removeComments: true
      },
      templateParameters(compilation, assets, options) {
        return {
          compilation: compilation,
          webpack: compilation.getStats().toJson(),
          webpackConfig: compilation.options,
          htmlWebpackPlugin: {
            files: assets,
            options: options
          },
          process,
        };
      },
      nodeModules: process.env.NODE_ENV !== 'production'
        ? path.resolve(__dirname, '../node_modules')
        : false
 }),

    # 打包一个应用

    • 启动成功后就有了第一个界面，但是如果我们有一个现成的网站，可以直接放进里面，相当于electron只是一个容器，通过webview的形式展示外链的内容。
    • 在main.js文件中修改winURL为我们的地址，在build的文件夹替换自己的icon，npm run build就打包好了。

    提示

    注意mac版本必须在mac系统才能打包，windows只能打包exe

    // const winURL = process.env.NODE_ENV === 'development'
