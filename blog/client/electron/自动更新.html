    # 自动更新

    • 在使用一款app经常会看到提示有新版本更新，客户端也不另外，electron开发的客户端自动更新用的是electron-updater插件,官方也有一个autoUpdater的api但是不好用。
    • 自动更新实现的方式有很多种，原理都差不多，这里说两种，一种是依赖于github检测更新后，手动下载安装更新。一种是依赖于自己的服务器，这种会自动检测更新下载安装。没有自己的服务器资源可以选择github。

    # github更新

    • github更新需要在github创建一个仓库来发布release版本，每次打开客户端，都会调用github api来获取最后一次发布的版本，再打包的时候在主进程声明一个变量记录打包的版本，打开客户端时对比两个版本，如果不是最新版就打开浏览器指向最新github release，然后再手动下载安装，比较节省资源，缺点是github速度很慢。
    • 打包后将.exe or .dmg .pk文件上传到github发布一个新的release版本。
    • github会提供一些api来管理自己的仓库，或者和自己的产品集成。
    • github-api文档
    //主进程

const version =1.02 //版本号以v+数字形式，每次更新版本也需要更新
const lastReleasesUrl="https://api.github.com/repos/xukeler/kloud-window/releases/latest";  //获取github版本号对比
const downUrl="https://github.com/xukeler/kloud-window/releases";  //打开github release
function checkVerion(){
  axios.get(lastReleasesUrl)
  .then(function (response) {
    
    var string=response.data.name.replace(/[A-z]/gi,'');
    var currentV=parseFloat(string);
    console.log(version<currentV)
    if(version<currentV){
      const child = new BrowserWindow({ parent: win, modal: true, show: false })
      child.loadURL('index.html')
      child.once('ready-to-show', () => {
        child.show()
      })
      dialog.showMessageBox({  //弹出一个窗口提示更新
        type: 'info',
        title: '发现新版本',
        buttons: ['Yes', 'No'],
        message: '      发现新版本，更新了很多功能，是否去下载最新的版本？      ',
        checkboxChecked: false
      }, (res) => {
        if (res === 0) { // if selected yes
          shell.openExternal(downUrl)
        }
      })
    }
  })
  .catch(function (error) {
    console.log(error);
  });

};
 checkVerion() //在createWindow之后调用

    # Server更新

    • Server更新需要将打包的文件放到服务器上，会多一个latest.ymml，mac是latest-mac.ymml文件，记录的是打包版本的版本好，打包日期，需要连同打包好的客户端一起放到服务器。
    • 使用electron-updater,在createWindow之后检查更新，有几个监听事件，来丰富更新的操作。
    • 注意在mac端自动更新是需要代码签名的，否则下载了也不会触发自动更新。
    • win下载完成后确定就会关闭开始安装更新，更新之后会自动覆盖原来的版本。
    //主进程
const { autoUpdater } = require('electron-updater');
function updater () {
  autoUpdater.autoInstallOnAppQuit=false;
  autoUpdater.setFeedURL('https://kloud.cn/')
  // 下面是自动更新的整个生命周期所发生的事件
  autoUpdater.on('error', function(message) {
    
  })
  // 获取版本信息
  autoUpdater.on('checking-for-update', function(message) {
    
  })
  // 即将进行更新
  autoUpdater.on('update-available', function(message) {
    
  })
  // 已是最新版
  autoUpdater.on('update-not-available', function(message) {
    
  })

  // 更新下载进度事件 progressObj.data.percent === 100  下载完成，
  // mac端会进行安装
  // win端不会自动安装而是走到 update-downloaded
  autoUpdater.on('download-progress', function(progressObj) {
    // sendUpdateMessage('downloadProgress', progressObj)
  })
  // 更新下载完成事件
  autoUpdater.on('update-downloaded', function(event) {
  	// 用于通知渲染进程下载完成（不通知直接关闭更新也可以，用户体验不好而已）
    dialog.showMessageBox({
      type: 'info',
      title: 'update',
      buttons: ['Yes',"No"],
      message:event.version+ '版本下载完成，是否马上安装？     ',
    }, (res) => {
      if (res === 0) { // if selected yes
        autoUpdater.quitAndInstall()
      }
    })
    
  })
  // 检测是否有更新
}
  win.webContents.on('dom-ready', (event) => {   //DOm加载好就检查更新
    autoUpdater.checkForUpdates();
    updater()
  })
