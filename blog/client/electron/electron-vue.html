    # electron-vue

    • electron-vue官方文档。
    • electron官方文档。
    • electron是一款使用javaScript开发跨平台客户端(mac window linux)的工具,electron-vue是结合了vue来跨平台开发，也能够使用vuex vue-router axios等其它插件，和正常开发vue项目一样。
    • electron由Node.js+Chromium+Native APIs构成。在里面也能使用node.js的api,和Os系统原生的api，Chromium内置的一个浏览器，用来显示前端界面。
    • electron分为两个进程，一个Main主进程和Renderer渲染进程,Main进程主要通过Node.js、Chromium和Native APIs来实现一些系统以及底层的操作，比如创建系统级别的菜单，操作剪贴板，创建APP的窗口等。Renderer渲染进程主要是渲染前端界面，一些node Native的api也能在渲染进程调用。

    # 安装electron-vue

    • 注意在vue init需要下载依赖可能会报错，因为一些依赖需要访问外国的地址，所以要给yarn | npm设置淘宝镜像。

npm install -g yarn

//添加 electron_mirror "https://npm.taobao.org/mirrors/electron/"

    # 目录结构

    • .electron-vue是electron的开发配置。
    • build是打包输出目录。
    • dist是将渲染界面打包的dist。
    • src包含main主进程的配置文件index.js，和开发配置文件index.dev.js。renderer包含渲染进程的目录，vue的开发目录都在这里面。
    • static静态资源存放目录。

    # 配置icon

    • 客户端都会有自己的安装图标，快捷方式图标，win的安装图标格式是.icon，mac的图标格式.icns，有一个默认的图标，可以替换成自己。尺寸要求256*256以上。
    • win还需要配置一个左上角的图标,大小64*64，在主进程的index.js中配置。
    • 由于的electron的配置项过多，多翻官方文档。
      mainWindow = new BrowserWindow({ 
    icon: require('path').join(__dirname, '/static/small.ico'), //配置小图标
    fullscreen: false,  
    show: false ,
    useContentSize: false,
    width:900,
    height:600,
    center:true,
    minWidth:900,
    minHeight:600,
  })

    # 打包程序

    • 打包用到了electron-builder插件，可以通过这个插件实现在windows和mac打包。打包mac只能在mac系统。
    • windows打包目前没有出什么意外情况，直接npm run build 就在build目录生成一个.exe文件。直接安装就好了。
    • 打包可能会遇见一个错误，提示从github下载一个文件失败，需要手动下载这个文件，然后放到指定的文件夹。win的文件夹C:\Users*****\AppData\Local\electron\Cache,mac文件夹~/Library(资源管理器)/Caches/electron/。

    • mac可能会遇见一个错误Command failed: /usr/bin/perl /private/tmp/t-4F3xVm/1-dmgProperties.pl，问题就是electron-builder的版本低了，需要升级到最新版本。
    npm install electron-builder@latest -D
    1
    • mac有几种打包程序格式pkg dmg zip等，打包成pkg dmg zip需要在package.json中配置。
    • 打包完成后就可以在build目录看到.exe或.pkg .dmg文件。
