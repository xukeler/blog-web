    # 多端小程序开发踩坑

    我们公司的小程序是用'uni-app跨端开发的，五端一套代码(qq小程序，微信小程序，百度小程序，头条小程序，飞书小程序)，uni-app不支持飞书，但是飞书可以编译头条的代码，头条和飞书公用，多端一套代码就会产生很多问题，这些问题基本都是平台不兼容导致的，很让人头疼，反馈了官方也不解决。

    # qq小程序

    • 获取光标问题textarea组件的blur方法无法获取光标位置
    • 获取接口返回的cookie,qqSet-Cookie是大写
    • 层级穿透问题
      • 在qq端存在一个非常严重的bug,textarea的层级在最顶层，在一个可以滚动的页面，滚动时textarea里面的内容可以显示到固定定位元素的上面，改z-index也没有用。
      • 建议textarea比较少就和div交替显示。
      • 比较多建议用webview实现

    # 百度小程序

    • v-for问题，百度端不支持在 html 标签内的 v-for=“item in number（5）的遍历，必须是数组才行。
    • webview 不支持长按保存图片。
    • 获取接口返回的cookie,百度Set-Cookie是大写
    • 数据绑定问题，百度端在dom上通过data绑定数据，属性名自动转换为小写，微信小程序支持大小写。
    <!-- 在百度小程序获取就是key -->
<view class="container" data-Key="123">

</view>

    # 飞书小程序

    • 飞书小程序的开发者工具调试经常出问题，所以要备份一个稳定版本。
    • 飞书小程序的图片错位问题，必须给图片的容器设置一个宽度width:100%。
    • webview 不支持长按保存图片。
    • 获取接口返回的cookie,飞书没有Set-Cookie只有cookies

    # 头条小程序

    • webview 不支持长按保存图片。
    • 获取接口返回的cookie,头条set-cookie是小写

    # 微信小程序

    • 获取接口返回的cookie,微信Set-Cookie是大写

    # ios端

    • textarea padding 问题,ios端 textarea会自带一个padding，二期无法去掉，安卓端没有

    # android端

    • replaceAll方法,安卓webview不支持js replaceAll这个方法，ios支持

    # webview刷新问题

    webview的url每一次刷新，都会在嵌入webview的h5的history历史记录加一层，导致点击小程序原生返回实际是h5的后退

    # 开发者工具问题

    除了微信的开发者工具，其它端的开发者工具不要随便升级，升级后可能就用不了，例如百度端，飞书端
