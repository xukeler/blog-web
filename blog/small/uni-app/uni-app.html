    # uni-app简介

    • uni-app是使用vue语法跨端开发的一个框架，一套代码开发，发布到多个平台，例如安卓App，ios App,微信小程序，百度小程序，支付宝小程序，头条小程序，普通h5,QQ小程序等平台，最大的特点就是使用vue语法结合uni-api开发，对于经常使用vue开发的能够快速上手。
    • 开发需要结合hbuildX IDE来运行到不同端进行测试，微信小程序测试设置好Ide路径,直接就能打开，其它端的测试在运行之后会有一个unpackage dist dev的文件夹，找到对应的问件在对应的Ide打开就可以测试。
    • 发布就在hbuildx的IDE选择发行对应的端，输入对应的名称和APPid就会在在对应的Ide打开，然后本地生成unpackage dist目录下有个build文件件将对应代码上传，发布审核。
    • 官网
    • hbuildX

    # 创建项目

    • 创建项目用hbuildX编辑器，在文件新建项目选择uni-app项目，里面会有一些内置好的模板项目，根据需要创建，填好项目名称，本地代码存储路径点击创建就完成了。

    # 项目结构

    • pages.json整个项目的全局配置，设置页面路径，窗口样式导航栏，tabbar等。

      • globalStyle设置项目的状态栏，导航条，标题，窗口背景色等。

        • navigationBarBackgroundColor导航栏背景颜色
        • navigationBarTextStyle导航栏标题颜色及状态栏前景颜色，仅支持 black/white
        • navigationBarTitleText导航栏标题文字内容
        • navigationStyle导航栏样式，仅支持 default/custom。
        • backgroundColor下拉显示出来的窗口的背景色
        • backgroundTextStyle下拉loading的样式，仅支持dark / light.
        • enablePullDownRefresh是否开启下拉刷新
        • onReachBottomDistance页面上拉触底事件触发时距页面底部距离，单位只支持px
        • backgroundColorTop顶部窗口的背景色
        • backgroundColorBottom底部窗口的背景色
        • titleImage导航栏图片地址
        • pageOrientation横屏配置，屏幕旋转设置，
        • animationType窗口显示的动画效果
        • animationDuration窗口显示动画的持续时间
        • usingComponents引用小程序组件

      • pages配置应用由哪些页面组成，接受一个数组，默认数组第一个元素为初始入口页，不需要写后缀。

        • path配置页面路径。
        • style和pages.json中的globalStyle一样的配置，会将全局配置覆盖调。

    • manifest.json是应用的配置文件，用于指定应用的名称、图标、权限等,在HbuildX是可视化的配置，对于开发小程序就是配置es6转es5代码压缩，检查域名，代码样式自动补全等配置。对于App的开发配置项要多一些。

    • package.json增加uni-app扩展结点，实现自定义条件编译平台。

    • vue.config,js vue2的项目配置文件，具体配置参考vue vue.config.js配置。

    • uni.scss一个全局的scss样式文件，不需要import引入，就可以在其它scss代码中使用。

    • App.vue uni-app的页面入口文件，和vue不同的是这里不能编写视图内容,也就是template里面的内容。

    • main.js是uni-app的入口文件，主要作用是初始化vue实例、定义全局组件、使用需要的插件如vuex。

    # 调试项目

    • 调试在hbuildX的顶部菜单栏选择运行，运行到小程序模拟器，再选择要运行的小程序平台，前提是其它平台的开发者工具也要安装好，注册小程序,在hbuildX填好各端开发者工具的安装路径。
    • 微信小程序点击运行之后会直接打开微信小程序的编辑器，其它端是将代码编译到根目录unpackage>dist>dev生成各端的代码包,在各端的开发者工具打开代码包编译即可。
    • 在hbuildx修改代码，其它开发者工具也会重新编译，如果觉得hbuildX用的不方便，可以用使用hbuildX创建项目，开启调试，代码修改使用其它的编辑器，效果是一样的。

    # 运行项目

    • hbuildX编辑器导入代码包
    • 点击编辑器顶部->工具->设置->运行配置->小程序运行配置 ，配置4端小程序开发者工具安装地址(暂不支持飞书)
    • 点击编辑器顶部运行，运行之后微信qq百度会自动打开开发者工具，头条飞书需手动打开开发者工具，在五端开发者工具导入项目，选择unpackage->dist->dev目录下的各端代码包
      • 运行到微信
      • 运行到头条
      • 运行到百度
      • 运行到qq

    # App.vue 应用生命周期

    • App.vue里面配置整个应用的生命周期，和全局变量，定义在globalData里面的变量是全局的，通过getApp().globalData获取，这样其实就可以不用使用vuex。
    • 在App.vue也能使用methods。
    • App.vue没有界面上的内容。
    • onLaunch初始化完成时触发（全局只触发一次）。
    • onShow从后台到前台显示就会触发。
    • onHide从前台到后台就会触发。
    • onError出现报错时触发。
    • onPageNotFound页面不存在监听函数。
    • onThemeChange监听系统主题变化。
    • 应用的生命周期函数只在App.vue页面监听有效。
    <script>
	export default {
		globalData:{
				isBaidu:false,
				name:'xukeler'
		},
		onLaunch: function() {
			console.log('初始化')
		},
		onShow: function() {
			console.log('进入前台')
		},
		onHide: function() {
			console.log('进入后台')
		},
		onError() {
			console.log("出现报错")
		}

	}
</script>

    # Page 页面生命周期

    • 页面的生命周期相对于应用生命周期增加了几个，但是常用的就那几个，更多查看官方文档。
    • Page其实就是路由页面，在page.json里面注册，里面可以像vue开发一样正常使用watch监听，computed计算属性，以及methods局部组件等vue特性。
    • onLoad监听页面加载，其参数为路由传递的数据，参数类型为个对象。
    • onShow监听页面显示。页面每次出现在屏幕上都触发。
    • onReady监听页面初次渲染完成。注意如果渲染速度快，会在页面进入动画完成前触发。
    • onHide监听页面隐藏。
    • onUnload监听页面卸载，一般在这个阶段清除全局计时器。
    • onPullDownRefresh监听用户下拉动作，一般用于下拉刷新。
    • onReachBottom监听页面滚动到底部的事件。
    • onShareAppMessage监听用户点击右上角分享。

    TIP

    注意事件的兼容性，有的只在相应的平台生效。

    <template>
	<view>
		<view class="">
			{{count}}
		</view>
		<view class="">
			{{countText}}
		</view>
		<view class="content">
			<button type="default" @click="addCount">增加</button>
		</view>
	</view>
</template>

<script>
	export default {
		data() {
			return {
				count:0
			}
		},
		watch:{
			count(){
				console.log("值改变了")
			}
		},
		computed:{
			countText(){
				return this.count+'个人'
			}
		},
		methods: {
			addCount(){
				this.count+=1;
			}
		},
		onLoad() {
			console.log("加载完成")
		},
		onShow() {
			console.log("显示")
		},
		onHide() {
			console.log("隐藏")
		},
		onPullDownRefresh() {
			console.log("下拉")
		},
		onShareAppMessage() {
			console.log("分享")
		},
		onReady() {
			console.log("准备好了")
		},
		onUnload() {
			console.log("页面卸载")
		}
	}
</script>

    # 组件

    • 虽然能够使用vue的特性开发，但是页面的html结构需要遵守uni-app规范，而不是web开发的div span的html标签结构。
    • uni-app提供了一些原生的组件，包含视图容器，表单，媒体组件等，直接在页面使用即可，使用需要注意不同平台的差异性，详细的查看官方文档。
    • 自定义组件其实就是自己封装的组件，需要放到components目录下，自己封装的组件声明周期只有vue的那几个声明周期函数created mounted updated beforeUpdate beforeDestroy destroyed,使用自定义组件和vue一样，import引入，在components注册。

    # 页面通信

    • 页面通信有多种方式
    • uni.$emit uni.$on uni.$off通过事件器实现。
    • 利用本地存储或者cookie实现。
    • 使用vuex实现。
    • 路由传参实现。

    # 路由跳转

    • 路由跳转是和微信小程序差不多的，包括传参方式，传递的参数会在onLoad声明周期中获取到。
    • 常用的uni.navigateTo uni.switchTab uni.navigateBack uni.redirectTo。
    • 都接收一个对象参数，对象包含url跳转路径和参数，success成功回调，fail失败回调,complete结束回调。
    • 比较简单具体参考官方文档

    TIP

    需要注意路由跳转不能一直使用uni.navigateTo，层级大概为10层左右，需要结合uni.navigateBack使用。

    # 语法结构

    • 语法和vue基本一致
    • 使用@click @tap，绑定事件，v-bind绑定属性，
    • v-if v-else-if v-else v-show条件渲染
    • v-for列表渲染。:key绑定。

    # 网络请求

    • 发送请求使用uni.request()方法，这个属于uni-app api的范畴了，需要传递一个对象，包含属性和回调函数。
    • url请求路径（域名要加入白名单）。
    • data请求的参数。
    • header设置请求头。
    • method请求类型。
    • timeout 超时时间，单位 ms。
    • responseType设置响应的数据类型。合法值：text、arraybuffer。
    • success请求成功回调，返回请求结果。
    • fail请求失败回调。
    • complete请求结束回调。

    # 条件编译

    • 条件编译是用特殊的注释作为标记，在编译时根据这些特殊的注释，将注释里面的代码编译到不同平台。
    • 在解决不同平台的兼容性会经常用到。
    • js html css都可以用。
    • #ifdef #endif 仅在某平台存在。
    • #ifndef #endif除了某平台均存在(可以使用||不能使用&&)。
    • MP-WEIXIN微信平台。
    • MP-ALIPAY支付宝平台。
    • APP-PLUSApp平台。
    • H5H5平台。
    • MP-BAIDU百度平台。
    • MP-TOUTIAO头条平台。
    • MP-QQQQ平台。
    		<!-- #ifdef  MP-WEIXIN -->
		<view class="content">
			MP-WEIXIN
		</view>
		<!-- #endif -->
        <!-- #ifdef  MP-BAIDU -->
		<view class="content">
			MP-WEIXIN
		</view>
		<!-- #endif -->
