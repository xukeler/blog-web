# 云闪付小程序

  • 最近开发一个需求，入驻云闪付小程序。调研了一下云闪付的开发文档，官方文档写的内容不多，提取有效内容和正常理解的小程序不一样，云闪付小程序本质是封装的webview打开h5，小程序就是h5。
  • 官方有一个sdk和一个封装的button ui组件，sdk是用来实现前端的api能力,button组件是获取用户信息的入口，和微信小程序登录类似。
  • 技术文档
  • 管理后台

# 开发

  • 开发还是使用vue 系列工具，官方提供了一个开发者工具，创建项目其实就是创建了一个vue 2的一个初始项目，和自己单独创建没什么区别，我们的项目有服务端渲染，所以这个开发者工具用不了(不支持)。

  • 在管理后台注册账号，必须企业认证之后才能创建小程序，认证创建小程序之后，需要设置小程序首页地址，也就是h5的首页地址，设置域名白名单，保存扫描测试二维码就可以体验。

  • 开发是用复用现有的项目，减少开发成本，所以要区分是不是云闪付环境，云闪付支持通过ua判断。


let isCloudpay=userAgent.toLowerCase().includes("unionpay")

# 登录

  • 登录通过云闪付button组件实现，点击button触发click事件，返回code再通过接口绑定账号。
  • button组件支持几种能力，目前自用到手机号授权。
    • scope.mobile手机号授权。
    • scope.auth实名授权
    • scope.bank银行卡信息授权
    • scope.face人脸授权
  • button组件使用之前先引入sdk，初始化后使用，因为是复用项目所以要异步局部引用。

# 异步加载sdk初始化

  • upsdk.setTitleStyle设置导航栏样式。
  • backBtnVisible设置导航栏左侧返回按钮是否显示。
loadJS("https://open.95516.com/s/open/js/upsdk.js", () => {
        upsdk.pluginReady(() => {
          this.readyState = true;
          upsdk.setTitleStyle({
            backBtnVisible: "1" // 可选，左侧返回按钮是否显示。'0'不显示，'1'显示，不传或空则默认显示
          });
        });
      });

// 异步加载js
export const loadJS = (src, callback) => {
  var script = document.createElement("script");
  var head = document.getElementsByTagName("head")[0];
  var loaded;
  script.src = src;
  if (typeof callback === "function") {
    script.onload = script.onreadystatechange = function() {
      if (
        !loaded &&
        (!script.readyState || /loaded|complete/.test(script.readyState))
      ) {
        script.onload = script.onreadystatechange = null;
        loaded = true;
        callback();
      }
    };
  }
  head.appendChild(script);
};

# 设置标题

  • 云闪付小程序(h5),每次路由跳转导航栏都需要有一个标题，所以每个页面都需要设置，如果不设置就显示默认的title，如果过长就不太美观，在用户体验上来说也不是很好。
  • 设置标题在开发的过程中发现安卓和ios有一定的区别。
    • 安卓直接设置标题document.title生效，ios不生效。
    • ios通过sdk修改。
let routeObj={
  "index": "首页",
  "list": "我的项目",
  "edit-id": "编辑",
  "preview-id": "预览",
  "result-id": "发布",
  "share-id": "分享",
}
if (this.routeObj[name]) {
  upsdk.setNavigationBarTitle({
    title: this.routeObj[name]
  });
}

# 手动分享

  • 手动分享用sdk的upsdk.shareSinglePlugin方法。
  • 分享出去之后，点击分享再返回云闪付shareUrl需要是用云闪付的，参数要加密处理。
  • 如果不用返回云闪付，shareUrl就可以随意写了，能打开就行。
        let obj = {
          toLink: encodeURIComponent("https://www.baidu.com"),
          encryptAppId: "xxxxxxxxx"
        };
        let params = new Base64().encode(JSON.stringify(obj));

        upsdk.shareSinglePlugin({
          title: this.projectData.title,
          desc: `我刚刚制作了一份问卷”，需要您的回答，您的意见非常重要。请速帮忙回答。`,
          shareUrl: `https://base.95516.com/s/wl/WebAPP/helpAgree/page/help/shareRutineHelp.html?params=${params}`, // 默认为当前页面链接即location.href
          shareId: type, // 详细见备注
          picUrl:
            "https://www.xxxxx.com/static/images/rspd_share.png",
          shareType: "1", //'1':链接分享，'2':文字分享，'4'：图片分享， 详细见备注
          shareData: {
            // 详细见备注
            content: "", //可选，当shareType为2时，必传
            image: "iVBORw…AAElFTkSuQmCC" //图片的base64数据，可选，仅当shareType为4时，传
          },
          fail: code => {
            if (code == "01") {
              upsdk.showFlashInfo({
                msg: "应用未安装"
              });
            }
            // code: '01':对应渠道的应用未安装，例如微信未安装，或该渠道调用失败
            // code: '02':检查shareType,shareaId是否正确
          }
        });
      }
