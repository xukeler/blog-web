    # 开发前准备

    • 开发微信小程序首先要注册一个账号注册地址。

    • 注册完成之后登陆进管理后台，可以看到里面的一些功能设置。

    • 开发一个小程序需要一个appId，这个appId在开发中的开发设置中可以找到。

    • 然后开始下载微信的开发者工具，下载地址
    • 下载完成后安装，打开开发者工具（需要微信扫描登陆），选择小程序，设置小程序的名称，项目目录，appid，也可以选择云开发，点击创建就生成了一个小程序项目。

    # 目录文件介绍

    • 小程序的初始项目目录。
    • pages是所有的页面组件目录，里面包含的每一个页面都有四个文件js（页面逻辑代码），json（页面属性配置），wxml（html代码），wxss（css样式）。
    • utils是一个存放公共js方法的目录。
    • app.js是全局js逻辑文件，在任何独立页面的js文件都能访问app.js里面方法。
    • app.json中进行全局的配置，例如导航栏的文字颜色，背景颜色，标题，tabbar的设置，都通过app.json进行配置。
    • app.wxcss全局的css样式设置。
    • project.config.json开发配置文件，例如es6转es5,和开发者工具里面的详情->本地配置保持一致。
    • sitemap.json配置小程序页面被微信索引。

    # app.json常用配置

    • 详细配置参考地址
    • 全局配置
    • pages用来配置页面路径信息，不需要写后缀名。
    • window用来配置小程序导航栏和状态栏窗口背景色等。
    • tabBar配置底部tab导航栏。
    • networkTimeout各类网络请求超时时间默认是60000毫秒。
    • debug开始debug模式方便用于调试，会将页面注册，页面路由数据更新打印出来。
    • usingComponents声明自定义组件为全局自定义组件或者第三方组件。
    • permission小程序接口权限设置,详细参考地址

    # 页面配置

    • 每个单独的页面都会有四个文件.wxml，.wxss，.json，.js，页面单独配置需要在json文件中配置，和app.json的区别就是一个是总的设置，一个是局部单独的设置。配置选项和app.json全局配置一样。
    • 小程序打包体积不能超过2Mb。

    # 页面生命周期函数

    • 每个页面都会有一个生命的周期从加载显示到隐藏销毁一个完整的流程，这些生命周期都包含在page（{}），里面，包括每个页面的data（类似于vue里面data）也包含在page（{}里面。
    • 新建的page页面在js文件中通过调用page({})方法传入一个对象，对象包含生命周期函数等配置。
    • data页面所有的动态数据绑定都来自于data，类似于vue的data。
    • onLoad，页面加载时触发，只触发一次，类似于vue的created。
    • onShow页面显示时触发。
    • onReady页面渲染完成时触发。类似于vue的mounted。
    • onHide页面隐藏或者切换到后台是触发。
    • onUnload页面卸载时触发，类似于vue的destoyed。
    Page({
  data:{
    value:'today'
  },
  onLoad(){
    console.log("初始化")
  },
  onReady(){
    console.log("准备好了")
  },
  onShow(){
    console.log('显示')
  },
  onHide(){
    console.log("隐藏")
  },
  onUnload(){
    console.log("页面卸载")
  },
})

    # 页面事件处理函数

    • 页面处理函数，是用户在当前页面的行为触发的，例如下拉刷新，滑动，分享，旋转
    • onPullDownRefresh监听用户下拉刷新函数，前提是需要在app.json中开启下拉刷新。
    • onReachBottom监听用户上拉触底事件。
    • onPageScroll监听用户滑动页面事件。
    • onShareAppMessage监听用户点击转发按钮
    • onResize页面尺寸改变触发。
    • onTabltemTap当前页面是tab页时，点击tab触发。
    • onShareTimeline用户点击右上角转发到朋友圈触发。
    • onAddToFavorites用户点击右上角收藏触发。
    Page({
  onPullDownRefresh(){
    console.log("下拉刷新")
  },
  onReachBottom(){
    console.log("触底")
  },
  onPageScroll(){
    console.log("页面滚动")
  },
  onShareAppMessage(){
    console.log("分享")
  },
  onResize(){
    console.log("页面尺寸改变")
  },
  onTabltemTap(){
    console.log("点击Tab")
  }
})

    # 数据绑定和事件绑定

    • 小程序的数据流是单向的即modal层到view层，定义在page data中的数据使用{}会在页面中渲染出来，修改数据使用this.setData()方法。
    • 在标签中动态绑定属性或者class，必须使用{{}}包裹起来，例如<view class="id{{}}"></view>
    • 事件可以通过catch bind来进行绑定，通过在标签bindtap绑定在page中定义的函数，点击的时候会触发,bind绑定的事件不会阻止冒泡，catch绑定的事件会阻止冒泡。
    • page里面定义的函数事件，默认第一个参数是event对象。
    • touchstart手指触摸动作开始触发。
    • touchmove手指触摸后移动触发。
    • touchcancel手指触摸动作被打断触发，如来电提醒，弹窗。
    • touchend触摸动作结束触发。
    • tap点击触发。
    • longpress触摸后超过350s之后离开触发。
    • longtap长按触发。
    <view bindtap="begin">点击</view>
<view catchtap="begin">点击</view>

Page({
    data:{
        value:"taoday"
    },
    begin(e){
      console.log(e)
        console.log(this.data.value)
    }

    # 小程序的生命周期

    • 小程序的生命周期是在app.js文件中定义的，在app.js中周期函数都包括在 App（{}）里面，这点是和页面周期函数有区别，app（{}）只能在app.js中调用，必须且只调用一次。
    • onLaunch监听小程序初始化。
    • onShow监听小程序启动或者切换到前台。
    • onHide监听小程序切换到后台。
    • onError监听错误函数。
    • onPageNotFound监听页面不存在函数。
    App({
  onLaunch (options) {
    // Do something initial when launch.
  },
  onShow (options) {
    // Do something when show.
  },
  onHide () {
    // Do something when hide.
  },
  onError (msg) {
    console.log(msg)
  },
})

    # 路由跳转

    • 微信小程序有一个全局对象wx,这个就类似于web的window对象，里面封装了一些属性和方法，路由也包含在内。

    • wx.switchTab跳转到 tabBar页面，并关闭其他所有非tabBar 页面。

      • url需要跳转的 tabBar 页面的路径 (代码包路径),不能传递参数。
      • success成功的回调。
      • fail失败的回调。
      • complete跳转完成的回调,调用成功、失败都会执行。

    • wx.reLaunch关闭所有页面，打开到应用内的某个页面。

      • url需要跳转的 tabBar 需要跳转的应用内页面路径 (代码包路径),可以参数，和query传参一致，传递的参数在目标路由的onload生命周期函数的options参数中可以获取到。
      • success成功的回调。
      • fail失败的回调。
      • complete跳转完成的回调,调用成功、失败都会执行。

    • wx.redirectTo关闭当前页面，跳转到应用内的某个页面。但是不允许跳转到 tabbar 页面,重定向跳转之后，导航栏没有返回的按钮，没有历史路由信息。

      • url需要跳转的应用内非tabBar 的页面的路径(代码包路径),可以参数，和query传参一致，传递的参数在目标路由的onload生命周期函数的options参数中可以获取到。
      • success成功的回调。
      • fail失败的回调。
      • complete跳转完成的回调,调用成功、失败都会执行。

    • wx.navigateTo保留当前页面，跳转到应用内的某个页面。但是不能跳到tabbar页面。使用wx.navigateBack 可以返回到原页面。

      • url需要跳转的应用内非tabBar 的页面的路径(代码包路径),可以参数，和query传参一致，传递的参数在目标路由的onload生命周期函数的options参数中可以获取到。
      • success成功的回调。
      • fail失败的回调。
      • complete跳转完成的回调,调用成功、失败都会执行。
    <view data-in="123" class="text-{{index}}" bindtap="jump">跳转</view>


  jump(){
    //wx.switchTab
    wx.switchTab({
      url: '/pages/center/center',
      success(){
        console.log("成功调用")
      },
      fail(){
        console.log("调用失败")
      },
      complete(){
        console.log("调用结束")
      }
    })
    //wx.reLaunch
    wx.reLaunch({
      url: '/pages/center/center?name=Mink&age=18',
      success(){
        console.log("成功调用")
      },
      fail(){
        console.log("调用失败")
      },
      complete(){
        console.log("调用结束")
      }
    })
    //redirectTo
    wx.redirectTo({
      url: '/pages/logs/logs?name=Mink&age=18',
      success(){
        console.log("成功调用")
      },
      fail(){
        console.log("调用失败")
      },
      complete(){
        console.log("调用结束")
      }
    })
  },

    # 小程序常用语法

    • 小程序的语法和vue语法差不多，只是在局部会有一些差异，会vue那小程序的语法理解就方便很多了。
    • 数据绑定和vue一样，在page的data设置好，在组件内通过双花括号{{}}包裹的形式调用，组件内使用data属性也要加双花括号。
    • wx:if条件渲染，类似于v-if。
    • wx:for列表渲染，类似于v-for，只不过不需要在设置渲染的index和item，默认的就是index，item。也可以通过wx:for-index来指定下标名称，wx:for-item指定item名称，同样也可以嵌套渲染。wx:key指定唯一标识符。
    <view class="index-container">
<van-button type="primary" wx:for="{{array}}" wx:key="index">{{item.name}}</van-button>
<van-button wx:if="{{flag}}">{{flag}}</van-button>
 </view>


Page({
  data:{
    value:'today',
    flag:true,
    array:[
      {name:"1"},
      {name:"1"},
      {name:"1"}
    ]
  }
})
