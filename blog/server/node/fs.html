# node.js

  • node.js是运行在服务端的JavaScript,像我们开发经常会用到node的npm包管理工具，一些前端的开发框架也是使用node来开启本地的服务，node可以看做是js编写服务端的程序，也能实现后端接口的功能，还能够结合electron开发客户端程序。

# fs读取本地文件

  • 读取本地文件用到的是fs模块的readFile方法，接受三个参数，一个是要读取的文件路径，一个是文件的格式(也可以不指定)，最后是回调函数，回调函数接受两个参数，一个是err错误，一个是data读取到的文件内容。

TIP

需要注意的是 node的路径是两个反斜杠\

fs.readFile("C:\\TxDoc\562\\SpecificationRoot\\42228\\123.txt",(err,data)=>{
  if(err){
    throw new Error(err)
  }else{
    console.log(data)
  }
})

# fs复制文件

  • 复制文件到指定的文件夹，使用fs模块的copyFile方法，接收四个参数，一个是要被拷贝的文件名(包含完整路径)，一个是粘贴的地址(包含文件名),一个是mode用于拷贝的操作符，一般用COPYFILE_EXCL文件已经存在则粘贴失败，最后一个是回调函数，接受一个参数err，复制粘贴成功err就是null。
fs.copyFile("C:\\TxDoc\562\\SpecificationRoot\\42228\\123.txt","D:\\TxDoc\562\\SpecificationRoot\\42228\\123.txt", COPYFILE_EXCL, (err)=>{

# fs判断路径是否存在

  • 判断一个路径是否存在,使用fs模块的existsSync方法，只接受一个参数就是要被判断的路径，存在返回true，不存在返回false。

TIP

existsSync的异步方法exists已经弃用

if(fs.existsSync("C:\\TxDoc")){
    console.log("路径存在")

# fs监听文件修改

  • 实时监听文件的修改使用fs模块的watchFile或watch方法。
  • watch接受三个参数，一个是监听的文件名(完整路径),一个是监听选项，例如是否监听子选项，一个是回调函数接受两个参数，一个是eventType，eventType 是 rename 或 change一个是文件名。
  • watchFile一个是监听的文件名(完整路径),一个是监听选项(可以省略)，一个是回调函数,接收两个参数，一个是当前修改的状态，一个是之前修改的状态，一旦文件改变就会触发回调函数。
    fs.watch("C:\\TxDoc\562\\SpecificationRoot\\42228\\123.txt", (type, name) => {
          console.log("文件改变了")
    });
    fs.watchFile("C:\\TxDoc\562\\SpecificationRoot\\42228\\123.txt", (curr, prev) => {
          console.log("文件改变了")
    });
