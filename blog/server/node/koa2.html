    # koa2

    • Koa是一个新的 web框架，基于node实现的web框架，将内部的核心模块进行封装，koa2是第二个版本。
    • koa2是利用es7的async/await特性，减少回调地狱的麻烦。
    • koa的大部分功能依赖于中间件实现，本身不具备。
    • koa2包含四个模块
      • application应用模块。
      • context上下文模块。
      • request请求模块。
      • reponse 响应模块。
    • 官网;

    # application应用模块

    • 一个Koa Application（简称 app）由一系列 generator 中间件组成。按照编码顺序在栈内依次执行,这些中间件有处理路由的，有处理参数的，有处理缓存的等。
    • application是koa的实例化对象，简写为app，这个对象下包含一系列的方法和属性。
      • app.name应用名称。
      • app.env 默认是 NODE_ENV 或 "development"运行环境。
      • app.keys 签名的 cookie 密钥数组。
      • app.proxy 决定了哪些 proxy header 参数会被加到信任列表中。
      • app.listen()用于启动服务，监听端口。app.listen 实际是 http.createServer 的简单封装。
      • app.use()加载中间件。
      • app.on("error",(err,ctx)=>{})错误监听，可以在错误处理函数监听日志。

    # context上下文模块

    • Koa 的上下文对象context(简写ctx)封装了request与response对象至一个对象中，并提供了一些帮助开发编写业务逻辑的方法。
    • ctx对象属性和方法
      • request koa的请求对象。
      • response koa的响应对象。
      • req node.js的请求对象。
      • res node.js的响应对象。
      • app``koa实例对象
      • state推荐的命名空间，用于通过中间件传递信息和你的前端视图。
      • cookies.get通过options 获取 cookie name。
      • cookies.set通过options 设置 cookie name。
      • throw用来抛出一个包含.status 属性错误的帮助方法。
      • respond为了绕过 Koa的内置 response处理，你可以显式设置 ctx.respond = false。 如果您想要写入原始的 res对象而不是让Koa处理你的response，请使用此参数。
    • ctx会直接包含request与response属性和方法，也就是ctx.request.header和ctx.header功能是一样的。

    # request请求模块

    • request 对象是在node的 原生请求对象之上的抽象，提供了诸多对HTTP 服务器开发有用的功能。
      • request.header=设置请求头对象。
      • request.headers请求头对象
      • request.method请求方法
      • request.url设置请求 URL, 对url重写有用。
      • request.originalUrl获取请求原始URL。
      • request.origin获取URL的来源，包括 protocol 和 host。
      • request.href获取完整的请求URL
      • request.path获取请求路径名。
      • request.path=设置请求路径名
      • request.querystring根据 ? 获取原始查询字符串.
      • request.host获取主机
      • request.hostname获取主机名
      • request.charset获取请求字符集
      • request.query获取解析的查询字符串
      • request.protocol返回请求协议,https 或 http
      • request.ip请求远程地址

    # response响应模块

    • response 对象是在 node 的原生响应对象之上的抽象，提供了诸多对HTTP服务器开发有用的功能。
    • response.header响应头对象。
    • response.status获取响应状态
    • response.body获取响应主体
    • response.set设置响应头
    • response.get获取响应头字段值
    • response.append添加响应头
    • response.redirect重定向

    # 中间件

    • 中间件官方一点的解释是指连贯整个Koa应用程序，并共享资源（上下文）的独立插件。
    • 共享通过content上下文对象。
    • 连接通过next()函数,中间件默认只执行第一个，需要连续执行，必须使用async await。
    • 中间件不能够单独使用，需要依附平台，只负责处理，不负责输入和输入。类似于流水线，从输入开始处理的每个环节都可以看做中间件。
    • koa的中间件是通过app.use插入处理的流程。
    • koa的中间件处理为先进后出模型，也就是洋葱模型或回形针模型。
    const Koa=require("koa2")
const app=new Koa();
const fun1=async (ctx,next)=>{
    console.log(1)
    await next()
    console.log(1.1)
}
const fun2=async (ctx,next)=>{
    console.log(2)
    await next()
    console.log(2.2)
}
const fun3=async (ctx,next)=>{
    console.log(3)
    await next()
    console.log(3.3)
}
app.use(fun1)
app.use(fun2)
app.use(fun3)
app.listen(3000)

// 输出

// 2
// 3
// 3.3
// 2.2
